* Python虚拟机
1) CPython
2) PyPy，用Python实现的虚拟机
3) Jython，用Java实现的虚拟机
4) IronPtython，用C#实现的虚拟机
5) RPython，带静态类型的Python

** 虚拟机运行机制
1) 完成模块（源码或字节码）的加载和链接
2) 将源码编译为字节码，写入内存供CPU读取
3) 从内存中读取字节码并执行，结束后将字节码回写到硬盘（文件后缀为.pyc或.pyo）

后续再次执行该脚本时，虚拟机会先检查字节码文件是否和源码一致（通过修改时间），一致则直接执行字节码。

* 包机制
Python代码按照模块(module)划分，一个模块可以是一个文件或一个包含一个或多个子模块的目录。而模块同时可以视作包(package)

** 模块检索
~str, len 和 Exception~ 等存在于全局作用域，在Python中被称为builtin scope；其他模块则通过 ~import~ 语句引入。

*** import语句
1) import语句可导入如下对象
   1) 模块文件(.py文件)
   2) C或C++扩展（已编译为共享库.so或DLL文件）
   3) 包（包含多个模块）
   4) 内建模块（使用C编译并已链接到Python解释器中）
2) 运行原理
   - Python解释锡会根据 ~sys.path~ 和 环境变量 ~PYTHONPATH~ 中定义的本地目录路径，按顺序依次检索模块，而 ~sys.path~ 默认第一元素为空字符串，即表示运行目录。
   - 运行目录：执行程序时程序所在的目录，固定不变。
   - 当前工作目录（Current Working Directory）：可通过环境变量等修改，作为读写本地文件系统中文件/目录的当前目录。
   - 导入的模块会以模块名和模块对象对的形式保存到 ~sys.modules~ 中，因此对于多次导入同一模块时，默认情况下仅第一次导入生效。
   - import实际上会调用__import__导入模块。

    ~site~ 模块控制着模块检索路径设置的方式，Python虚拟机初始化时，会自动被引用。？？？？？

** 模块和包的区别
包为一个或多个模块的集合，一般压缩打包为tarball的文件形式，内含
1) 依赖关系说明
2) 将文件复制到标准的模块检索路径的说明（复制到 ~sys.path~ 下的某个路径）
3) 编译说明（内含平台相关的源码，需要按需编译）

* .pyc、.pyo和.pyd文件
.pyc为.py的字节码(byte code)文件，内部保存的是PyCodeObject的序列化信息，不是二进制代码。
.pyd为Python的动态链接库文件。

注意：
1) .pyc会保存在源码相同的目录下；
2) .pyc的内容与Python版本有关，因此Python2.4生成的.pyc不能在Python2.5上运行；
3) .pyo是优化版的.pyc文件，体积更小。

*** 自定义上下文管理器
上下文管理器就是遵循了上下文管理协议（Context Management Protocol）的类， 即实现了 ~__enter__~ 和 ~__exit__~ 方法的对象。

#+BEGIN_EXAMPLE python
import sys
import cx_Oracle

class OracleContext(object):
    def __init__(self, uri, schema, passwd, err={}):
        self.connstr = '%s/%s@%s' % (schema, passwd, uri)
        self.err = err
        self.err['value'] = None

    def __enter__(self,):
    """进入with语句块时会调用上下文管理器实例的__enter__函数，并将其返回值绑定到as子句后的变量上。
  
    返回值为列表或元组时，as子句为 as (a,b,c) 进行解构
    """
        self.conn = cx_Oracle.connect(
            self.connstr, encoding='utf-8', nencoding='utf-8')

        return self.conn

    def __exit__(self, ex_type, ex_value, ex_traceback):
    """退出with语句块时会掉用上下文管理器实例的__exit__函数，无论with语句块是否抛出异常。

    若with语句块中没有抛出异常，或者执行break/continue/return语句，那么以None作为参数调用context_manager.__exit__(None, None, None)
    若抛出异常则会以sys.exc_info()的作为参数调用context_manager.__exit__(*sys.exc_info())

    Args:
        ex_type: 异常类型
        ex_value: 异常实例
        ex_traceback: 异常发生时的调用堆栈信息

    Returns:
        当with语句块中抛出异常时，返回True表示忽略异常，False则重新抛出异常
    """
        if ex_type:
             self.err['value'] = str(ex_value)
        
        if self.conn:
            if ex_type:
                self.conn.rollback()
            else:
                self.conn.commit()

            self.conn.close()

        return True

# 使用
rows = []
with OracleContext('','','') as conn:
    cursor = conn.cursor()
    cursor = cursor.execute('select 1 from dual')
    rows = cursor.fetchall()

# 虚拟机会将上述语句翻译为
rows = []
context_manager = OracleContext('', '', '')
exit = type(context_manager).__exit__
conn = type(context_manager).__enter__(context_manager)
try:
    cursor = conn.cursor()
    cursor = cursor.execute('select 1 from dual')
    rows = cursor.fetchall()
except:
    if not exit(context_manager, *sys.exc_info()):
        raise
else:
    exit(context_manager, None, None, None)
#+END_EXAMPLE

*** 通过contextlib模块定义上下文管理器
contextlib模块提供一些帮助方法和修饰器去定义用于with语句块的对象。

1) contextmanager修饰符
#+BEGIN_EXAMPLE python
from contextlib import (
    contextmanager,
)
import cx_Oracle

@contextmanager
function db_transaction(connstr):
    """yield前的代码作为__enter__方法，yield后的代码作为__exit__方法。
    with语句块的异常将通过yield语句传递到生成器函数中。
    """
    conn = cx_Oracle(connstr)
    try:
        yield conn
    except:
        conn.rollback()
    else:
        conn.commit()
    finally:
        conn.close()      

rows = []
with db_transaction('') as conn:
    cursor = conn.cursor()
    cursor = cursor.execute('select 1 from dual')
    rows = cursor.fetchall()
#+END_EXAMPLE

** 自定义迭代器
迭代器就是实现了next方法的类
#+BEGIN_EXAMPLE python
class Text(object):
    def __init__(self, text):
        self.__cursor = 0
        self.__text = text

    def __iter__(self,):
        return self

    def next(self,):
        if self.__cursor == len(self.__text):
            raise StopIteration()
        else:
            i = self.__cursor
            self.__cursor += 1
            return self.__text[i]

for char in Text('abcd'):
    print(char)
#+END_EXAMPLE

* 修饰器
面向切面编程，对原有函数进行增强。

注意： 模块加载时马上执行修饰器生成新函数。

#+BEGIN_EXAMPLE python
from functools import wraps

def add1_missing_attrs(func):
    """定义个普通的修饰器
    """
    def inner(*args, **kwargs):
        result = func(*args, **kwargs)
        return result + 1

    return inner

def add1_leaving_attrs(func):
    """由于装饰器实际是高阶函数，因此原函数的__name__,__doc__和__module__等函数特性会被外层函数替换掉， 通过functools.wraps复制原函数特性到外层函数特性。
    """
    @wraps(func)
    def inner(*args, **kwargs):
        result = func(*args, **kwargs)
        return result + 1

    return inner    

def addx(x):
    def tmp(func):
        @wraps(func)
        def inner(*args, **kwargs):
            return x + func(*args, **kwargs)
        
        return inner
    
    return tmp

@add1_missing_attrs
def add2(n):
    return n + 1

@add1_leaving_attrs
def add3(n):
    return n + 2

@addx(4)
def add4(n):
    return n
#+END_EXAMPLE

* Lambda函数
Lambda函数又称为匿名函数，Python中的Lambda函数有以下限制：
1) 函数体中必须仅包含一个表达式，表达式的返回值作为函数的返回值；
2) 闭包/周围全局作用域(surrounding global scope)采用延迟绑定。
#+BEGIN_EXAMPLE python
def bug():
    return [lambda x: x * i for i in xrange(0, 5)]

def bingo():
    return [lambda x, i=i: x * i for i in xrange(0, 5)]

Q：为什么在类型A中调用 ~super(A, self).__init__()~ 却调用了类型C的 ~__init__~ 方法？

    """1. API为/users/:users/status对应的SDK为SDK().users('john').status。
       2. API为/users/status对应的SDK为SDK().users.status。

    def __init__(self, path=''):

    def __getattr__(self, seg):

        else:
            return SDK('%s/%s' % (self.__path, seg))

        return self.__path

    def __repr__(self,):
        return '<SDK %s>' % self.__path

from pprint import pprint

class ModelMetaclass(type):
    """自定义类型类。
    必须继承type，并且约定俗成名称以Metaclass结尾。
    """
    def __new__(cls, name, bases, attrs):
        """创建类型时会调用__new__方法来创建。
        
        Args:
            cls: 准备创建的类对象
            name: 类名称
            bases: 元组类型，父类列表
            attrs: 字典类型，属性和方法

        Returns: 类型
        """
        if name == 'Model':
            return type.__new__(cls, name, bases, attrs)
        
        mappings = dict()
        for k, v in attrs.iteritems():
            if isinstance(v, Field):
                mappings[k] = v
                pprint('Found mapping: %s ==> %s' % (k, v))

        for k in mappings.iterkeys():
            attrs.pop(k)

        attrs['__table__'] = name
        attrs['__mappings__'] = mappings
        return type.__new__(cls, name, bases, attrs)


class Model(dict):
    __metaclass__ = ModelMetaclass

    def save(self,):
        tpl_sql = 'insert into %s(%s) values(%s)'
 
        fields = []
        params = []
        args = []
        for k, v in self.__mappings__.iteritems():
            fields.append(v.name)
            params.append('?')
            args.append(getattr(self, k, None))

        sql = tpl_sql = % (self.__table__, ','.join(fields), ','.join(params))
        pprint('SQL: %s' % sql)
        pprint('ARGS: %s' % str(args))


class Field(object):

* 日志
官网： https://docs.python.org/2/library/logging.html

** 概念
日志类型：诊断日志、审计日志。
诊断日志：记录与应用操作相关的日志，如异常日志（用于线上排查问题）
审计日志：为商业分析而记录的日志，如Apache的访问记录日志（统计PV、UV和最热资源、出错率等信息）

Logger，调用的接口，自身带日志级别特性，一个Logger可对应0~N个Handler；
LogRecord，将从Logger得到的日志传递到Handler；
Handler，用于配置日志输出地点和保存时间，自身带日志级别特性，一个Handler对应一个Formatter；
Filter，用于配置输出日志的级别（debug,info,warn,error和critical）；
Formatter，用于配置日志的输出格式。
默认输出格式： ~<日志级别>:<Logger实例名称>:<日志消息内容>~ ，如 ~WARNING:root:warn message~ 。

日志级别
1) notset，最低日志级别；
2) debug，详细信息，只有开发调试时需要关注；
3) info，证明事件按预期工作，突出强调应用的运行过程；
4) warn/warning，发生可容忍的错误，但需要修复；
5) error，严重错误，但程序可以继续运行，需要马上修复；
6) critical/fatal，严重错误，程序已经不能继续运行。

logging日志记录流程
1) 判断当前Logger的日志级别是否可用，不可用则不输出日志，并流程结束；
2) 创建LogRecord，判断是否符合注册到当前Logger的Filter条件，不符合则不输出日志，并流程结束；
3) 将LogRecord交给当前Logger的Handler（若没有配置Handler则进入第四步），判断Handler的日志级别是否可用，若可用则判断是否符合注册到该Handler的Filter条件，若符合则输出日志；
4) 若当前Logger是否含父日志对象，有则将LogRecord传递给父日志对象，递归执行第一步直至到根日志对象。

若产生了LogRecord但整个流程都没有找到Handler进行判断Handler的日志级别是否可用，若可用则判断是否符合注册到该Handler的Filter条件等后续操作，那么会向sys.stdout输出提示信息。

        # time.clock()获取当前程序占用的CPU时间（time.time会包含其他程序占用的CPU时间）
        start = time.clock()
        ret = func(*args, **kwargs)
        end = time.clock()
        logger.debug('function %s running with args %s and kwargs %s cost %s secs.'
                    , func.__name__
                    , str(args)
                    , str(kwargs)
                    , end-start)

        return ret

    return inner


@record_time
def add(a, b):
    return a + b

* 单元测试

** 通过unittest模块进行单元测试

*** 概念
*TestCase* 测试用例，包含setUp、run和tearDown；
*TestSuite* 针对某个功能的测试用例集合；
*TestLoader* 将TestCase加载TestSuite中；
*TestRunner* 执行TestSuite或TestCase，并将执行结果保存到TestResult中。根据保存形式分为TextTestRunner、HTMLTestRunner和XMLTestRunner；
*TestFixture* 执行测试前后的工作，包括setUp、tearDown、setUpClass和tearDownClass。

# run_test.py
# -*- coding: utf-8 -*-
import os
import sys
import unittest

def gen_test_suite_partially():
    """选择部分TestCase加入TestSuite
    """
    from test_case_git import TestGit

    test_suite = unittest.TestSuite()
    test_suite.addTest(TestGit('test_a'))

    return test_suite

def gen_test_suite_from_class():
    """选择一个类的TestCase加入TestSuite
    """
    from test_case_git import TestGit

    test_suite = unittest.TestSuite()
    test_suite.addTest(unittest.makeSuite(TestGit))

    return test_suite

def gen_test_suite_fully():
    """从目录下加载TestCase到TestSuite.
    """
    case_path = os.getcwd()
    test_suite = unittest.defaultTestLoader.discover(case_path, pattern='test_case*.py')

    return test_suite

def gen_text_runner():
    """使用默认的TextTestRunner.
    """
    runner = unittest.TextTestRunner(verbosity=2)  #  verbosity控制输出的执行结果的详细程度，可为0，1，2，其中0最简单，1是默认值，2最详细
    return runner

def gen_html_runner():
    """使用HTMLTestRunner.
    """
    from HTMLTestRunner import HTMLTestRunner
    fp = open('test_result.html', 'wb')
    runner = HTMLTestRunner(stream=fp, title='Test Report', description='Test Description')
    return runner

def gen_xml_runner():
    """使用XMLTestRunner.
    Jenkins只认识xml格式的测试报告。
    pip install xmlrunner
    """
    from xmlrunner import XMLTestRunner
    runner = XMLTestRunner(output='report')
    return runner

if __name__ == '__main__':
   num = sys.argv[1]
   runner_type = sys.argv[2]

   test_suite = None
   if num == 1:
       test_suite = gen_test_suite_partially()
   else:
       test_suite = gen_test_suite_fully()       

   if runner_type == 1:
       runner = gen_text_runner()
   elif runner_type == 2:
       runner = gen_html_runner()
   else:
       runner = gen_xml_runner()

   runner.run(test_suite)

测试
测试
