* 列表中的key属性

** diff算法
   1) 广度优先同层比对；
   2) 从根往子组件逐层比对；
   3) 发现一层不同则不再往下一层比对，从不同的层级开始重新生成节点渲染。

** 为什么兄弟节点需要设置key
   https://reactjs.org/docs/reconciliation.html#recursing-on-children

** Controlled Components
   元素的值由React管理，用户修改输入内容时必须修改React Element的状态然后由React控制组件的变化。
   注意：若设置为null后，元素的内容就根据用户输入变化，而不受React管控。
   #+begin_example js
   class LoginForm extends React.Component {
     constructor(props){
       super(props)
       this.state = {account: "",
                     password: ""}
       this.handleAccountChange = this.handleAccountChange.bind(this)
       this.handlePasswordChange = this.handlePasswordChange.bind(this)
     }

     handleAccountChange(e){
       this.setState({account: e.target.value})
     }

     handlePasswordChange(e){
       this.setState({password: e.target.value})
     }

     handleSubmit(e){
       e.preventDefault()
       console.log(this.state.account)
       console.log(this.state.password)
     }

     render(){
       return (
         <form onSubmit={this.handleSubmit}>
           <label>
             Account: <input type="text" value={this.state.account} onChange={this.handleAccountChange} />
           </label>
           <label>
             Password: <input type="password" value={this.state.password} onChange={this.handleSubmit} />
           </label>
           <input type="submit" value="submit" />
         </form>
       )
     }
   }
   #+end_example

** HOC中使用Forwarding Ref
   #+begin_example js
   function logProps(Component){
     class LogProps extends React.Component{
       componentDidUpdate(prevProps){
         console.log(prevProps)
         console.log(this.props)
       }

       render(){
         const {forwardedRef, ...rest} = this.props

         return <Component forwardedRef={forwardedRef} ...rest />
       }
     }

     return React.createRef((props, ref) => (
       <LogProps {...props} forwardedRef={ref} />
     ))
   }

   const FancyButton = props => (
     <button ref={props.forwardedRef}>
       {props.text}
     </button>
   )

   const el = <LogFancyButton ref={ref} text="Hello!"/>

* Context API
  提供一种方式在组件树中传递数据，而不用自上而下逐层传递．适用于不同层级的组件共享变量值，如current authenticated user,locale,UI Theme,data cache等多组件需要的全局属性则适用于通过该方式传递。

** API说明
   #+begin_example js
   /* 创建上下文对象，订阅上下文对象的组件会读取最近一个Provider提供的值
    * @param defaultValue {Any} - 上下文中的默认值
    */
   const MyContext = React.createContext(defaultValue)

   /* 允许其子元素将订阅上下文实例内容的变化。
    * 注意：1. Provider可嵌套，元素获取最近的一个Provider提供的值
    *       2. 当Provider的value属性值发生变化时，订阅上下文变化的组件将无条件重新渲染，即使其父组件在shouldComponentUpdate中返回false。
    * @param value {Any} - 作为上下文的值
    */
   <MyContext.Provider value={/* some value */} />

   /* Class.contextType
    * 用于在Class Component中的所有生命周期函数中使用上下文值
    */
   class FancyButton extends React.Component{
     componentDidMount(){
       let value = this.context
     }
     componentDidUpdate(){
       let value = this.context
     }
     componentWillUnmount(){
       let value = this.context
     }
     render(){
       let value = this.context
     }
     // static contextType = MyContext 启用public class fields syntax时可用这个语法设置
   }
   FancyButton.contextType = MyContext // 传统设置方式

   /* 用于在Function Component中使用上下文值
    */
   <MyContext.Consumer>
     {value => /* render something based on the context value */}
   </MyContext.Consumer>
   #+end_example

   #+begin_example js
   // app.js
   import React from "react"
   import {theme, ThemeContext} from "theme-context"
   import ThemeButton from "theme-button"

   class App extends React.Component {
     constructor(props){
       super(props)
       this.state = {theme: theme.dark}

       this.toggleTheme = this.toggleTheme.bind(this)
     }

     toggleTheme(){
       this.setState(state => {theme: state.theme == theme.dark ? theme.light : theme.dark})
     }

     render(){
       return (
         <div>
           <ThemeContext.Provider value={this.state.theme}>
             <ThemeButton onClick={this.toggleTheme}/>
             <ThemeContext.Consumer>
               {theme => <div style={{color: theme.foreground}}>Hello!</div>}
             </ThemeContext.Consumer>
           </ThemeContext.Provider>
         </div>
       )
     }
   }
   #+end_example

* Error Boundaries
  React 16引入该特性，用于解决问题组件不会导致整个系统不可用的问题。
  Error Boundaries的处理流程：
  1) 捕获当前元素及其子元素的渲染函数、生命周期函数和构造函数中的异常
  2) 记录错误日志信息
  3) 渲染一个降级UI

  注意：Error Boundaries不获取以下异常
  1) Event Handlers
  2) Asynchronous code
  3) Server side rendering
  4) Errors thrown in the error boundary itself(rather than its children)

** 自定义Error Boundary Component
   #+begin_example js
   import React from "react"

   class ErrorBoundary extends React.Component {
     constructor(props){
       super(props)
       this.state = {hasError: false}
     }

     static getDerivedStateFromError(error){
       // 只要实现这个函数或componentDidCatch函数，那么该组件即为Error Boundary Component。
       // 该函数用于在捕获到异常时返回降级UI用的state，然后在render函数中根据state值显示渲染降级UI。
       return {hasError: true}
     }

     componentDidCatch(error, info){
       // 只要实现这个函数或static getDerivedStateFromError函数，那么该组件即为Error Boundary Component。
       // 该函数用于捕获异常时记录异常日志信息
       logger.error(error, info)
     }

     render(){
       return this.state.hasError ? (<h1>Something went wrong.</h1>) : this.props.children
     }
   }

   <ErrorBoundary>
     <FancyButton/>
   </ErrorBoundary>
   #+end_example

** 未捕获异常的默认行为
   As of React 16, errors that were not caught by any error boundary will result in unmounting of the whole React component tree.

** 异常消息堆栈
   异常消息堆栈包含如下信息：
   1) 调用链中的组件名称
      这里通过 ~Function.name~ 或 ~Function.displayName~ 获取组件名称，因此对于IE11以前的浏览器则需要使用 function.name-polyfill(https://github.com/JamesMGreene/Function.name) 处理。
   2) 组件所在文件名

** 事件和异步代码引发的异常
   由于Error Boundaries用于捕获导致组件渲染异常的问题，而事件和异步代码的异常在组件渲染阶段并不响应组件进行渲染，因此需要采用JS的 ~try/catch~ 处理。

*** 生命周期函数详解
   1) constructor
      1) 用于初始化内部状态和绑定方法
         #+begin_example js
         constructor(props){
           this.state = {type: 0}
           this.handleClick = this.handleClick.bind(this)
         }
         #+end_example
      2) 唯一可以直接修改state的地方
   2) getDerivedStateFromProps
      React 16.3 引入的新的生命周期函数，用于根据外部属性生成内部状态并作为函数返回值返回，然后会将返回值自动合并到组件的状态上。
      1) 为类静态方法
      2) 用于替代之前的componentWillReceiveProps
      3) 方法签名
         #+begin_example js
         /*
          * @param {object} nextProps - 新传入来的props
          * @param {object} prevState - 之前的state，在方法体内 this.state === prevState 返回true
          * @return {object} - 新的state，若不产生新的state则返回null
          */
         static getDerivedStateFromProps(nextProps, prevState){
           const {type} = nextProps
           if (type != prevState.type){
             return {type,}
           }
           return null
         }
         #+end_example
      4) 如果父组件的state进行了更新，那么其子组件也会触发它们的getDerivedStateFromProps方法。
      5) 使用该方式时必须通过constructor初始化state，否则会在控制台显示警告信息。
      6) 使用条件
         1) 无条件地更新状态
         2) 当props和state不匹配时更新state
      7) 尽量不使用：维护两者状态一致性会增加复杂度
         1) 直接在render中使用通过计算props的值更容易保持一致性，而不是另外设置到state上。
      8) 每次render都会调用到
      9) 典型场景：表单控件获取默认值
   3) render
      组件唯一必须定义的生命周期函数，用于定义DOM结构的地方。
      1) 方法签名
         #+begin_example js
         /*
          * @return {ReactElement|null}
          */
         render(){
         }
         #+end_example
   4) componentDidMount
      这个方法在刷新页面前触发的，因此在其中调用 ~this.setState~ 触发2次render，用户看到的效果也是页面被刷新一次。但会引起性能问题。
      - 用处：在这里发起异步请求等IO操作
   5) shouldComponentUpdate
      用于自定义某些操作是否需要更新UI，以便作性能优化。返回true为更新UI，false为不更新UI且中止当前流程。
      1) 在页面render之前调用，state已更新，DOM未更新但可访问
      2) 方法签名
         #+begin_example js
         /*
          * @param {object} nextProps - 最新的props
          * @param {object} nextState - 最新的state
          * @return {boolean} - true继续渲染，false中止当前组件渲染流程，但不影响子组件的更新渲染流程
          */
         shouldComponentUpdate(nextProps, nextState){}
         #+end_example
      3) 不能在方法体内调用 ~this.setState~
      4) 典型场景：获取render之前的DOM状态
   6) getSnapshotBeforeUpdate
      React 16.3 引入的新的生命周期函数
      1) 可以读取未更新时的DOM信息，但无法操作DOM，然后将信息传递给componentDidUpdate。
      2) 方法签名
         #+begin_example js
         /*
          * @param {object} prevProps - 变化前的props
          * @param {object} prevState - 变化前的state
          * @return {any}
          */
         getSnapshotBeforeUpdate(prevProps, prevState)
         #+end_example
      3) 示例-获取更新前的滚动条位置
         #+begin_example js
         class ScrollingList extends React.Component {
           constructor(props){
             super(props)
             this.listRef = React.createRef()
           }

           getSnapshotBeforeUpdate(prevProps, prevState){
             if (prevProps.list.length < this.props.list.length){
               const list = this.listRef.current
               return list.scrollHeight - list.scrollTop
             }

             return null
           }

           render(){
             return (
               <div ref={this.listRef}>
                 {this.list.map(item => <div key={item}>{item}</div>)}
               </div>
             )
           }

           componentDidUpdate(prevProps, prevState, snapshot){
             if (snapshot !== null) {
               const list = this.listRef.current
               list.scrollTop = list.scrollHeight - snapshot
             }
           }
         }
         #+end_example
   7) componentDidUpdate
      #+begin_example js
      /* @param {object} prevProps - 之前的props
       * @param {object} prevState - 之前的state
       * @param {any} dataFromGetSnapshortBeforeUpdate - getSnapshotBeforeUpdate函数的返回值
       */
      componentDidUpdate(prevProps, prevState, dataFromGetSnapshortBeforeUpdate)
      #+end_example
      1) 典型场景：页面需要更新props变化重新获取数据
   8) componentWillUnmount
      用于释放资源。
      1) 注意：在初始化阶段若当前组件和子组件的 ~componentDidMount~ 发生异常，也会触发 ~componentWillUnmount~ 的执行。
      2) 方法签名
         #+begin_example js
         componentWillUnmount(){}
         #+end_example

** Integrating with DOM Manipulation Plugins
   由于React并不知道DOM变化，而是自身维护一份Virtual DOM并映射到DOM中。若外界直接修改DOM，那么React再次通过Virtual DOM映射DOM时将蒙圈。

        }

        componentWillUnmount(){
          this.$el.component("destroy")
          this.$el.off("click")
        }

        render(){

        }
      }

* React.PureComponent
  为React 15.3 引入的新特性，用于取代之前的PureRenderMixin，减少不必要的render操作。
  适合用于一些纯展示状态的组件上。

** 原理
   通过shallowEqual排除不必要的render
   1) 通过 ~Object.keys(state)~ 和 ~Object.keys(props)~ 比较prev和next的长度是否一致；
   2) prev和next是否拥有相同的key(值不进行比较)；
   3) prev和next指向的地址是否一致。

   也就是仅对比第一层，当上述三个条件之一不符合则触发渲染，否则中止当前组件的渲染流程(子组件继续)。

** 与shouldComponentUpdate共存时
   规则
   1) 存在shouldComponentUpdate时则以shouldComponentUpdate结果为准；
   2) 没有shouldComponentUpdate时则按PureComponent的shallowEqual结果为准。

** 实例
   当父组件调用render函数时就会触发子组件调用render函数，当实现 PureComponent 后若传递给子组件的props内存地址不变，则不会触发子组件的render函数。

  用于解决IO问题
  过去：在render函数中发起异步请求将在下一个JS事件循环中才会执行，此时渲染已经执行完毕，因此拿到数据也无法触发重新渲染。

   处理逻辑：
   1) 在render函数中发起一个异步请求获取数据（这个异步请求的结果会被缓存起来）；

** React的SyntheticEvent池化陷阱
   由于React默认采用池化技术重用SyntheticEvent，从而减少SyntheticEvent对象的创建，且每次使用后会先置空其中的所有属性，再返还给资源池。（官方文档：https://facebook.github.io/react/docs/events.html#event-pooling）
   因此在通过debounce或throttle等延迟执行方案执行事件回调函数时，在回调函数中访问SyntheticEvent对象，那么能有可能已经被回收了。

*** 解决方法1
    1. 避免在回调函数中访问SyntheticEvent对象；
    2. 通过外部变量存储SyntheticEvent对象的属性，供回调函数使用。

*** 解决方案2
    通过SyntheticEvent的persist方法，设置该SynthenticEvent对象不返回资源池。

状态管理框架，基于flex架构。

注意： *整个应用的所有组件的状态都保存在一个store中*

优点：
1) 确定性状态视图(与纯组件结合时启用确定性视图呈现)
2) 事务状态
3) 状态管理与IO和副作用隔离
4) 应用状态使用单一信息源
5) 在不同组件间轻松共享状态
6) 事务遥测(自动记录操作对象)
7) 跟踪调试

特性：
1) Single Source of Truth
2) 可预测性
   state+action=new state
3) 纯函数更新store

** 核心概念
1) Store
   #+begin_example js
   /* 创建Store实例
    * @param {function} reducer - 纯函数
    * @return {Store}
    */
   const store = createStore(reducer)

   /* 从Store实例中获取当前状态
    */
   store.getState()
   /* 将一个action(如点击某按钮)分发给某个reducer
    * @param {Object} action - 中间件可修改action的类型，如使用react-thunk时，action接受Function类型
    * @return {Any} - action内容，若使用react-thunk则会thunk函数的返回值
    */
   store.dispatch(action)
   /* 订阅Store实例中状态的变化，变化发生时回调listener方法。
    * 回调方法中通过store.getState()获取Store中的数据，然后决定是否要调用组件的this.setState触发更新
    * @return {function} unsubscribe - 退订，在componentWillUnmout中调用来释放资源
    */
   store.subscribe(listener)

   store.replaceReducer
   #+end_example

*** 示例-计数器
    #+begin_example js
    import {createStore, bindActionCreators} from "redux"
    import {Provider, connect} from "react-redux"

    // redux
    const counter = (state = {count: 0}, action) => {
      switch(action.type){
        case "INCREMENT":
          return {count: state.count + 1}
        case "DECREMENT":
          return {count: state.count - 1}
        default:
          return state
      }
    }

    store = createStore(counter)

    let increment = () => {type: "INCREMENT"}
    let decrement = () => {type: "DECREMENT"}

    // react component
    class Counter extends React.Component {
      render(){
        // 调用connect后默认会将dispatch作为props传递给组件
        const {count, increment, decrement, dispatch} = this.props
        return (
          <div>
            <button onClick={decrement}>-</button>
            <span>{count}</span>
            <button onClick={increment}>+</button>
          </div>
        )
      }
    }

    // 通过connect连接
    /* 自动订阅store的状态变化，一旦变化则调用该方法，从Store中提取state作为传给component的props
     * 若connect函数中不传递该函数，则不订阅和不响应store的状态变化。
     * @param state - store.getState()的返回值
     * @param {any} [ownProps] - 高阶组件自带的属性，若函数中声明这个参数，那么当高阶组件自身的props发生变化也会触发该方法
     * @return {object}
     */
    const mapStateToProps = (state, ownProps) => {count: state.count}
    /* 生成包含action creator和dispatch两步的函数作为传给component的props
     */
    const mapDispatchToProps = (dispatch, ownProps) => bindActionCreators({increment, decrement}, dispatch)
    // 生成高阶组件
    const ConnectedCounter = connect(mapStateToProps, mapDispatchToProps)(Counter)
    //const ConnectedCounter = connect(null, mapDispatchToProps)(Counter)

    // 使用
    const App = props => (
      <Provider store={store}> // 作为整个系统的根节点
        <ConnectedCounter />
      </Provider>
    )
    #+end_example

** 异步Action
   1) 工作原理
      在Action和Dispatcher之间添加一个Middleware层，用于拦截异步Action和转发异步Action的内和最终产生的Action给Dispatcher。
      #+begin_example js
      export default function applyMiddleware(...middlewares) {
        return (createStore) => (reducer, preloadedState, enhancer) => {
          var store = createStore(reducer, preloadedState, enhancer);
          var dispatch = store.dispatch;
          var chain = [];

          var middlewareAPI = {
            getState: store.getState,
            dispatch: (action) => dispatch(action)
          };
          chain = middlewares.map(middleware => middleware(middlewareAPI));
          dispatch = compose(...chain)(store.dispatch);

          return {...store, dispatch}
        }
      }
      #+end_example

         function getList(){
           // 异步Action
           // 入参dispatch为Middleware传入，用于转发异步Action内产生的Action给Dispatcher
           // 必须返回Promise对象
           return dispatch => {
             dispatch({type: "BEGIN"})

             const promise = new Promise((resolve, reject) => {
               const doRequest = axios.get("....")
               doRequest.then(res => {
                 dispatch({type: "SUCCESS", data: res.data})
                 resolve(res)
               }, err => {
                 dispatch({type: "FAIL", data: {error: err}})
                 reject(err)
               })
             })
             return promise
           }
         }

         function reducer(state = {}, action){
           swith(action.type):
             case "BEGIN":
               return {
                 ...state,
                 fetchListPending: true,
                 fetchListError: false
               }
             case "SUCCESS":
               return {
                 ...state,
                 data: action.data
                 fetchListPending: false,
                 fetchListError: false
               }
             case "FAIL":
               return {
                 ...state,
                 fetchListPending: false,
                 fetchListError: true
               }
             default:
               return state
         }

** Scroll Restoration

单页面跳转时浏览器会保留滚动条的位置，可以通过自定义元素处理。
通用方案的基本准则：
1) Scrolling up on navigation so you don't start a new screen scolled to the bottom.
2) Restoring scroll positions of the window and overflow elements on "back" and "forward" clicks.

定义：
#+begin_example js
import React from "react"
import {withRouter} from "react-router"

const schema = {
  type: "array",
  items: {
    type: "object",
    properties: {
      pathname: {
        type: "string"
      },
      x: {
        type: "number"
      },
      y: {
        type: "number"
      }
    }
  }
}

class ScrollRestoration extends React.Component {
  render(){
    return null
  }

  componentDidUpdate(prevProps){
    const {location} = prevProps
    if (this.props.location.pathname !== location.pathname){
      this.positions = [{pathname: location.pathname, x: window.scrollX, y: window.scrollY}]
                       .concat(this.positions.filter({pathname} => pathname !== location.pathname))

      const currPosition = this.positions.find(({pathname}) => pathname === this.props.location.pathname)
      if (currPosition) {
        window.scrollTo(currPosition.x, currPosition.y)
      }
      else {
        window.scrollTo(0, 0)
      }
    }
  }

  componentDidMount(){
    this.positions = JSON.parse(sessionStorage.getItem("scroll-restoration") || "[]")
    const {location} = this.props
    const currPosition = this.positions.find(({pathname}) => pathname === location.pathname)
    if (currPosition) {
      window.scrollTo(currPosition.x, currPosition.y)
    }
  }

  componentWillUnmount(){
    sessionStorage.setItem("scroll-restoration", JSON.stringify(this.positions))
  }
}

export default withRouter(ScrollRestoration)
#+end_example

** Macro Task
   setTimeout, setInterval, requestAnimationFrame 发起的异步任务将被放置 Macro Task Queue。
   Macro Task Queue 可以存在多个。
   Macro Task Queue 的第一个任务就是入口JavaScript程序。

** Micro Task
   Promise, process.nextTick 发起的异步任务将被放置 Micro Task Queue
   Micro Task Queue 仅能存在一个。
   执行规律为：每个事件循环先执行完 Micro Task Queue 中所有任务，然后再执行 Macro Task Queue 的一个任务。

*** 坑
    1. ~math.format~ 第二个参数precision默认用于指定整体进行四舍五入后的有效数的个数，不是指保留小数位后多少位
       #+begin_src javascript
       math.format(0.7+0.1, 2) // 返回 '0.8'
       math.format(10.7+0.1, 2) // 返回 '11'
       math.format(10.7+0.1, 1) // 返回 '10'

       // 若指定为保留小数位后多少位，则需要将notation设置为fixed
       math.format(0.7+0.1, {notation: 'fixed', precision: 2}) // 返回 '0.80'       
       math.format(10.7+0.1, {notation: 'fixed', precision: 2}) // 返回 '10.80'       
       math.format(10.7+0.1, {notation: 'fixed', precision: 0}) // 返回 '11'       
       #+end_src

** JavaScript标准库Number方法
   1. ~Number#toFixed([precision: number = 0]): string~
      - 作用：将Number四舍五入为指定小数位的数字,若小数点后有效位数不足则用 ~0~ 补足.
      - ~[precision: number = 0]~ 取值范围 0 至 20, 若过大则会抛出RangeError.
   2. ~Number#toPrecision(precision: number): string~
      - 作用：当整数位数大于有效数时，将超出指定有效位部分转换为科学计数法表示，即 ~10000..toPrecision(4)~ 得到 ~'1.000e+4'~ ； 当整数位数+小数位数小于有效数，则采用四舍五入得到定点记数表示法表示，如 ~100.46..toPrecision(3)~ 得到 ~'100.5'~
      - ~[precision: number]~ 取值范围 1 至 21, 若过大则会抛出RangeError. 省略则会调用 ~toString()~ 方法。
   3. ~Number#toExponential(precision: number)~
      - 作用：将超出指定有效位部分转换为科学计数法表示，即 ~10000..toExponential(4)~ 得到 ~'1.000e+4'~
      - ~[precision: number]~ 取值范围 0 至 20, 若过大则会抛出RangeError. 省略则会调用 ~toString()~ 方法。
