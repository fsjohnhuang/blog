* 8种基本数据类型
  - byte, 1byte signed integer
    - default 0
    - min -128
    - max 127
    - 包装类 Byte
  - short, 2byte signed integer
    - default 0
    - min -32768
    - max 32767
    - 包装类 Short
  - int, 4byte signed integer
    - default 0
    - min -2,147,483,648
    - max 2,147,483,647
    - 包装类 Integer
  - long, 8byte signed integer
    - default 0L
    - min -9,223,372,036,854,775,808
    - max 9,223,372,036,854,775,807
    - 包装类 Long
  - float, 4byte IEEE 754 float
    - default 0.0f
    - 不能用于表示精确的值，如货币
    - 包装类 Float
  - double, 8byte IEEE 754 float
    - default 0.0d
    - 不能用于表示精确的值，如货币
    - 包装类 Double
  - boolean
    - default false
    - 包装类 Boolean
  - char, a 2byte Unicode character
    - default \u0000
    - min \u0000
    - max \uffff
    - 包装类 Character
    - 示例
      #+BEGIN_EXAMPLE java
      char uniChar = '\u039A'; // Unicode 字符表示形式
      char[] charArray = {'a', 'b'};
      #+END_EXAMPLE
  
  8种基本数据类型的包装类的属性
  - SIZE，返回数据类型所占的位数
  - MIN_VALUE，返回最小值
  - MAX_VALUE，返回最大值
  所有基本数据类型的包装类都继承Number
  静态方法
  - valueOf(primitive_value)，将基本数据类型转换为包装类
  实例方法
  - xxxValue()，从包装类实例中提取内部的基本数据类型变量值

  

  隐式类型转换： ~byte,short,char -> int -> long -> float -> double~ 
  显式强制转换需注意内容：
  1) 浮点数转整数会直接舍弃小数，而不是四舍五入
  2) 不能对boolean类型进行类型转换

* 局部变量、实例变量和静态变量
  1) 局部变量
     - 在线程栈上分配存储空间
     - 没有默认值，因此声明后必须经过初始化后才能使用
  2) 实例变量
     - 有默认值
       - 数值型变量为0
       - 布尔型变量为false
       - 引用类型变量为null
     - 变量值的初始化时机
       1) 声明时设置
       2) 构造方法中设置
  3) 静态变量
     - 在静态存储区分配存储空间
     - 有默认值
       - 数值型变量为0
       - 布尔型变量为false
       - 引用类型变量为null
     - 变量值的初始化时机
       1) 声明时设置
       2) 构造方法中设置
       3) 静态语句块中设置

** ~java.util.Date~

   - 实例方法

** 日期时间字符串格式化
   1) ~java.text.SimpleDateFormat~
      继承 ~java.text.DateFormat~ ，提供locale-sensitive的日期和格式化字符串之间的相互转换功能。
      - 日期时间转换为格式化字符串
      #+BEGIN_EXAMPLE java
      // 假定当前时区为东8(即GMT+8)
      // 因此所有时间的默认时区均为东8
      Date now = new Date(); // 假设日期时间为 2019-04-21 00:00:00
      SimpleDateFormat ft = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
      String gmt8 = ft.format(now); // 2019-04-21 00:00:00
      ft.setTimeZone(TimeZone.getInstance("GMT")); // 设置时区为0时区
      String gmt = ft.format(now); // 2019-04-20 16:00:00
      #+END_EXAMPLE
      - 将字符串解析为日期时间
        #+BEGIN_EXAMPLE java
        SimpleDateFormat ft = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        Date date = ft.parse("2019-02-02 00:02:00");
        #+END_EXAMPLE
   2) ~System.out.printf~
      以 ~%t~ 开头并以下面表格的一个字母结尾即可格式化日期时间
      | 转换符 | 说明                                            | 示例                             |
      |--------+-------------------------------------------------+----------------------------------|
      | c      | 全部日期和时间信息，和new Date().toString()一样 | 星期六 十月 27 14:21:20 CST 2007 |
      | F      | yyyy-MM-dd                                      |                                  |
      | D      | MM/dd/yy                                        |                                  |
      | r      | hh:mm:ss pm(12进制)                             | 02:25:51 下午                    |
      | T      | hh:mm:ss (24进制)                               | 14:25:51                         |
      | R      | hh:mm (24进制)                                  | 14:28                            |
      | B      | 月份全称                                        | January                          |
      | b      | 月份简称                                        | Jan                              |
      | A      | 星期全称                                        |                                  |
      | a      | 星期简称                                        |                                  |
      | C      | 年的前两位                                      |                                  |
      | y      | 年的后两位                                      |                                  |
      | m      | 两位数字的月份                                  |                                  |
      | d      | 两位数字的日期                                  |                                  |
      | e      | 一位数字的月份（前面不补零）                    |                                  |
      - 示例
        #+BEGIN_EXAMPLE java
        Date date = new Date();
        // 普通使用方式
        System.out.printf("%tR%n", date); // 显示14:28换行符
        // 使用索引
        System.out.printf("%1$s %2$tR %2$tT", "hi", date); // 显示hi 14:28 14:28:41没有换行符
        #+END_EXAMPLE
   3) ~String.format~
      采用和 ~System.out.printf~ 一致的格式化编码，但可以设置语言区域
      #+BEGIN_EXAMPLE java
      Date date = new Date();
      String str = String.format(Locale.US, "英语月份简称：%tb", date);
      String str = String.format("本地月份简称：%tb", date);
      #+END_EXAMPLE

* 修饰符
用于定义类、方法和变量，通常放在语句的最前端

** 访问控制修饰符
1) default 在同一个包内可见。使用对象：类、接口、方法、变量
2) public 公开。使用对象：类、接口、方法、变量
3) protected 对同一包内的类和所有子类可见。使用对象：方法、变量、内部类
4) private 在同一个类内可见。使用对象：方法、变量、内部类

*** 接口的默认访问修饰符
1) 变量默认为 ~public static final~
2) 方法默认为 ~public abstract~

** 非访问控制修饰符
1) final 用来修饰类、方法和变量
   1) 修饰类，类为最终类不能被继承
      1) 其下方法自动添加final修饰符
      2) 其下成员变量不会自动添加final修饰符
   2) 修饰方法，方法不能被子类重写(Override)
   3) 修饰变量，变量值不能被修改
      1) 变量为引用类型，那么变量指向的引用地址不能修改，但引用对应的内部属性可以被修改
         #+BEGIN_EXAMPLE java
         final Map<String, String> map = new HashMap<String, String>();
         map.put("one", "1");
         #+END_EXAMPLE
      2) 搭配static声明类变量作为常量使用
2) abstract 用来修饰抽象类和抽象方法
   1) 抽象类不能同时声明为final
   2) 抽象方法不能同时声明为final或static
   3) 若一个类包含抽象方法，那么该类也必须声明为抽象类
3) static 用来修饰类方法和类变量
4) synchronized 声明方法同一时间只能被一个线程访问
   - 示例 ~public synchronized void showDetails() {}~
5) volatile 修饰成员变量在每次线程访问时均强制从共享内存中重新读取值，并且当发生变化时强制线程将变化值回写到共享内存
6) transient 修饰变量不参与类的序列化

     // 枚举类Sex
     public enum Sex {
       MALE(1, "男性"), FEMALE(2, "女性"); // 枚举值MALE和FEMALE
  
       private int id;
       private String name;
       
       // 枚举的构造方法始终为private，因此可以省略访问控制符
       private Sex(int id, String name){
         this.id = id;
         this.name = name;
       }
  
       public int getId(){
         return this.id;
       }
  
       public void setId(int id){
         this.id = id;
       }
  
       public String getName(){
         return this.name;
       }
  
       public void setName(String name){
         this.name = name;
       }
  
       public static Sex getSex(int id){
         if (id == 1){
           return MALE;
         }
         else if (id == 2){
           return FEMALE;
         }
  
         return null;
       }
     }
     #+END_EXAMPLE
  3) 枚举实例可覆盖枚举类中定义的方法
     #+BEGIN_EXAMPLE java
     public enum Sex{
       MALE(1) {
         public int getId(){
           return this.id + 1;
         }
       },
       FEMALE(2) {
         public int getId(){
           return this.id + 2;
         }
       };
       

       protected int id;

       Sex(int id){
         this.id = id;
       }
       
       public abstract int getId();
     }
     #+END_EXAMPLE
  4) 枚举类可以拥有静态成员
     注意：枚举常量在静态成员之前构建，因此不能在构造方法中引用静态成员。若要初始化时修改静态成员值，可通过静态代码块处理。
     #+BEGIN_EXAMPLE java
     public enum Sex {
       MALE, FEMALE;
       public static int count = 1;
       private int c;
       
       // 在枚举常量构建后执行静态代码块
       static {
         for(Sex sex : Sex.values()){
           sex.c = count;
           count += 1;
         }
       }
     }
     #+END_EXAMPLE

* 面对对象编程

** ~super~ 和 ~this~
   ~super~ 用于显式访问父类成员
   ~this~ 用于显式访问当前类的成员

** 构造方法
   子类不会继承父类的构造方法，默认情况下会隐式调用父类的无参构造方法，但可通过 ~super~ 来显式调用父类的其它构造方法。
   #+BEGIN_EXAMPLE java
   class Super1 {
     Super1(){
       System.out.println("Super1 default constructor.");
     }
     Super1(int num){
       System.out.println("Super1 constructor with argument `num`.");
     }
   }
   class Child1 extends Super1{
     Child1(){
       super(10);
       System.out.println("Child1 default constructor.");
     }

     Child1(int num){
       System.out.println("Child1 constructor with argment `num`.");
     }
   }

   Super1 s1 = new Child1();
   Super1 s2 = new Child1(10);
   /* Super1 constructor with argument `num`.
    * Child1 default constructor.
    * Super1 default constructor.
    * Child1 constructor with argment `num`.
    */
   #+END_EXAMPLE

** 接口
   实现接口的抽象类可不实现接口的所有方法，否则则需要实现接口的所有方法。
   1) 接口和抽象类的区别
      1) 接口不能包含静态代码块和静态方法
      2) 一个类可实现多个接口，但只能继承一个抽象类
      3) 接口的方法不能有方法体，而抽象类中的非抽象方法将包含方法体
      4) 接口的成员变量修饰符只能为 ~public static final~ ，抽象类的成员变量修饰符和普通实现类一样
      5) 接口的成员方法修饰符只能为 ~public~
   2) 接口定义
      #+BEGIN_EXAMPLE java
      [public|private] interface <接口名称> [extends <其它接口名称>...]{}
      #+END_EXAMPLE
   3) 接口多继承
      接口支持多继承，即可通过 ~extends~ 继承多个父接口
      #+BEGIN_EXAMPLE java
      interface A{}
      interface B{}
      interface C extends A, B {}
      #+END_EXAMPLE
   4) 标记接口
      不包含任何属性和方法，仅用于表明它的实现类属于一个特定的类型拥有某个或某些权利。
   5) 接口可以拥有静态方法，一般作为工厂方法
   6) 接口可以拥有默认方法(Java8+)
      #+BEGIN_EXAMPLE java
      public interface Test{
        // 实现类可选择性地重写或沿用默认方法
        default void print(){
          System.out.println("test");
        }
      }
      #+END_EXAMPLE
      - 多接口方法冲突：若继承的多个接口定义了相同方法签名的方法
        - 若没有提供默认方法，那么派生类只需定义方法体，或定义为抽象类即可
        - 若存在默认方法，则派生类必须重写该方法
          #+BEGIN_EXAMPLE java
          interface A {
            default void print(){
              System.out.println("A");
            }
          }
          interface B {
            void print();
          }
          class C implements A, B {
            @Override
            public void print(){
              A.super.print(); // 调用接口的默认方法
            }
          }
          #+END_EXAMPLE

** 重写(Override)和重载(Overload)
   - 重写(Override)
     子类重新实现父类的签名相同的实例方法
     1) 子类方法不能抛出比父类方法范围更大的异常，仅能抛出父类方法声明的异常或其派生类
        #+BEGIN_EXAMPLE java
        class Super1{
          void echo throws IOException () {}
        }
        class Child1 extends Super1{
          // 编译时报错，因为子类方法抛出的异常范围比父类的更大
          void echo throws Exception () {}
        }
        #+END_EXAMPLE
     2) 子类方法的参数列表必须和父类的一致
     3) 子类方法的返回值类型要么和父类的一致，要么为父类方法返回值类型的派生类(Java5前必须和父类的一致)
     4) 子类方法的访问权限不能低于父类的
        1) 父类为public，那么子类必须为public
        2) 父类为default，那么子类可以为default或public
        3) 父类为protected，那么子类可以为protected、default或public
     5) 在子类方法中可通过 ~super~ 调用父类中被重写的方法
     6) 静态方法不存在重写，父类和子类的静态方法各自独立
   - 重载(Overload)
     在同一个类中多个名称相同但方法参数列表不同的实例或静态方法，重载方法的返回值类型可相同也可以不同
     1) 重载方法必须具有不同的参数列表，仅返回值类型不同是无法区分方法的

** 多态
   三个必要条件：
   1) 继承
   2) 重写
   3) 父类引用指向子类对象
      #+BEGIN_EXAMPLE java
      Super1 super1 = new Child1();
      #+END_EXAMPLE

** 注解
   1) ~@Override~ 指示该方法为重写父类的同名方法，那么在编译时会检查被标识的方法签名是否与父类签名一致，若不一致则编译失败。

** 泛型(generics/parameterized type)
   从Java5引入的特性，提供了编译时类型安全检查机制，防止运行时的执行与数据类型不符的非法操作。泛型本质为 *参数化类型* ，即为所操作的数据类型指定一个参数。
   注意点：
   1) 类型参数只能代表 *引用类型* 不能是原始类型

*** 泛型方法
    #+BEGIN_EXAMPLE java
    // 入参类型和返回值类型采用泛型
    public <E> E print(E a){
      System.out.println(a);
      
      return a;
    }

    // 多个泛型
    public <E, T> T forceCast(E a){
      return (T)a;
    }
    
    // 有界泛型，通过extends或implements设置泛型必须为某类或接口的派生类
    public <T extends Comparable<T>> T maximum(T x, T y){
      T max = x;
      if (y.compareTo(max) > 0){
        max = y;
      }

      return max;
    }
    #+END_EXAMPLE

*** 泛型类
    #+BEGIN_EXAMPLE java
    public class Box<T>{
      private T value;
      public T getValue(){
        return this.value;
      }
      public void setValue(T value){
        this.value = value;
      }
    }
    #+END_EXAMPLE

*** 类型限定符
    1) ~extends~ 用于限制为某类或接口的派生类
       对于继承多个类或接口则通过 ~&~ 表示，如 ~<T extends Comparable & Serializable>~
    2) ~super~ 用于限制为某类及父类、基类类型，如Object类型
*** 类型通配符
    1) ~?~ 用于对不关心应类型参数具体值的场景
       #+BEGIN_EXAMPLE java
       void printData(List<?> data){
         System.out.println(data.get(0));
       }
       #+END_EXAMPLE
    2) 带 ~extends~ 的有界泛型通配符，用于对不关心应类型参数具体值但需要限制为某类或接口的派生类
       注意：实现某接口也是用 ~extends~
       #+BEGIN_EXAMPLE java
       void printNumber(List<? extends Number> data){
         System.out.println(data.get(0) + 1);       
       }
       #+END_EXAMPLE
    3) 带 ~super~ 的有界泛型通配符，用于对不关心应类型参数具体值但需要限制为某类及父类、基类类型，如Object类型
       #+BEGIN_EXAMPLE java
       void printNumber(List<? super Number> data){
         System.out.println(data.get(0) + 1);       
       }
       #+END_EXAMPLE
   
*** 底层原理
    类型擦写(type erasure)：编译器在编译时会将所有类型参数信息擦除掉，在运行时不存在任何类型参数相关的信息。如 ~List<String>~ 在运行时只剩下 ~List~ ，这样就能兼容Java5之前版本的二进制类库，因此称Java的泛型为伪泛型。
    由于类型参数信息对JVM来说是不可见的，因此无法避免在运行时出现的类型转换异常的情况，但也不会出现真泛型中的类型膨胀问题。

    问题示例1——运行时Class类实例相相等
    #+BEGIN_EXAMPLE java
    ArrayList<String> list1 = new ArrayList<String>();
    list1.add("abc");
    ArrayList<Integer> list2 = new ArrayList<Integer>();
    list2.add(123);

    // 返回true
    System.out.println(list1.getClass() == list2.getClass());
    #+END_EXAMPLE
    
    问题示例2——运行时反射方法调用
    #+BEGIN_EXAMPLE java
    ArrayList<String> list1 = new ArrayList<String>();
    list1.add("abc");
    // list1.add(123); 编译失败
    Method m = list1.getClass().getMethod("add", Object.class);
    m.invoke(list1, 123); // 运行正常
    #+END_EXAMPLE
    
**** 显式和隐式声明类型变量    
     #+BEGIN_EXAMPLE java
     public class App{
       public static <T> T add(T x, T y){
         return y;
       }

       public static void main(String[] args){
         Integer i = App.add(1, 2); // 隐式声明类型变量为Integer
         Float d = App.add(1, 2.1); // 隐式声明类型变量为Float
         
         Integer i = App.<Integer>add(1, 2); // 显式声明类型变量为Integer
         Float d = App.<Float>add(1, 2.1); // 显式声明类型变量为Float
         
         // Integer i = App.<Integer>add(1, 2.1); // 编译报错
         Number n = App.<Number>add(1, 2.1);
       }
     }
     #+END_EXAMPLE

**** 类型擦写原理
     编译时将所有类型参数(parameterized type)中指定的类型转换为原始类型(raw type)，然后自动在取值位置添加类型转换代码。
     1) 原始类型，即最后在字节码中类型变量真正的数据类型；
     2) 转换原始类型的规则
        1) 没有类型限定符时，统一转换为Object；
        2) 附带 ~extends~ 类型限定符，统一转换为第一个基类或接口；
           #+BEGIN_EXAMPLE java
           public <T extends Comparable & Serializable> T get(){ return null; }
           // 转换为public Comparable get() { return null; }
           #+END_EXAMPLE
           *建议：将标记接口放在限制列表的最后*
        3) 附带 ~super~ 类型限定符，统一转换为Object。
     3) 自动添加类型转换代码
        #+BEGIN_EXAMPLE java
        public <T> T get(T a){ return a; }
        Integer a = <Integer>get(1);
        
        // 编译后的代码逻辑如下
        public Object get(Object a){ return a; }
        Object oA = get(1);
        Integer a = (Integer)oA;
        #+END_EXAMPLE

**** 限制分析
     1) 类型变量不能为8种基础类型和void
        因为类型擦写后的原始类型为Object，基础类型强转Object无效，因此必须使用它们的封装类才行。
     2) 编译期仅根据引用类型变量的类型参数作检查，而不管实际引用的对象的类型参数
        #+BEGIN_EXAMPLE java
        ArrayList list1 = new ArrayList<String>();
        list1.add("abc");
        list1.add(123);

        ArrayList<String> list2 = new ArrayList();
        list2.add("abc");
        list2.add(123); // 编译失败
        #+END_EXAMPLE
     3) 静态方法和属性不能引用类的类型参数
        由于类的类型参数是在类实例化时才能确定，若静态方法和属性引用类的类型参数，则无法确认类型参数的实际类型
        #+BEGIN_EXAMPLE java
        public class Basket<E>{
          public static void add(E e); // 编译失败
        }
        #+END_EXAMPLE
     4) 不支持类型参数的直接实例化，但可以通过反射实现
        #+BEGIN_EXAMPLE java
        public <T extends Comparable> T[] test(T[] a){
          // T m = new T(); 编译失败
          // T[] ms = new T[2]; 编译失败

          T[] ms = (T[])Array.newInstance(a.getClass().getComponentType(), 2);
          return ms;
        }
        #+END_EXAMPLE
     5) 重写含类型参数的方法
        #+BEGIN_EXAMPLE java
        class P<T>{
          private T val;
          public void set(T val){
            this.val = val;
          }
          public T get(){
            return this.val;
          }
        }

        class C extends P<Date>{
          @Override
          public void set(Date val){
            super.set(val);
          }
          @Override
          public Date get(){
            return super.get();
          }
        }

        /**
        * 编译后
        */
        class P{
          private Object val;
          public void set(Object val){
            this.val = val;
          }
          public Object get(){
            return this.val;
          }
        }

        class C extends P{
          public void set(Date val){
            super.set(val);
          }

          public Date get(){
            return super.get();
          }

          // 编译器自动生成桥接方法
          @Override
          public void set(Object val){
            this.set((Date)val);
          }

          // 编译器自动生成桥接方法
          // Object get() 和 Date get()仅返回值不同，而参数列表相同。在编译时被认为是相同的方法，会报重复定义。
          // 但JVM将返回值+参数列表作为方法签名，因此在运行时被认为是不同的方法。
          @Override
          public Object get(){
            return this.get();
          }
        }
        #+END_EXAMPLE
     6) 类型参数不能出现在 ~catch~ 中
        #+BEGIN_EXAMPLE java
        try{}
        catch(T e){}
        #+END_EXAMPLE
     7) 泛型类不能出现在 ~catch~ 中
        #+BEGIN_EXAMPLE java
        try{}
        catch(Problem<T> e){}
        #+END_EXAMPLE
     8) 泛型类不支持 ~instanceof~ ，除非用通配符
        #+BEGIN_EXAMPLE java
        ArrayList<String> sList = new ArrayList<String>();
        // if (sList instanceof ArrayList<String>) 编译失败
        if (sList instanceof ArrayList<?>) {
          // do something
        }
        #+END_EXAMPLE

*** 泛型的型变(type variance)——协变(convariance)和逆变(contravariance)
    型变(type variance)
    #+BEGIN_QUOTE
    Type variance refers to the techniques by which we can allow, or not allow, subtyping in our parameterized types.
    #+END_QUOTE
    假设现在存在 ~List<Orange>~ 和 ~List<Fruit>~ 两个类型，它们可以存在以下任一个关系：
    1) 不型变(invariance): ~List<Orange>~ 和 ~List<Fruit>~ 两者没有任何关系；
    2) 协变(covariance): ~List<Orange>~ 为 ~List<Fruit>~ 的子类型；
    3) 逆变(cotravariance): ~List<Fruit>~ 为 ~List<Orange>~ 的子类型；
    
    *子类(Subclass)* 和 *子类型(Subtype)* 不是一个概念，子类一定是子类型，但子类型不一定是子类。如 ~List<Orange>~ 是 ~List<? extends Fruit>~ 的子类型，但不是子类。

**** Java中的协变
     #+BEGIN_EXAMPLE java
     List<? extends Object> oList;
     List<String> sList = new LinkedList<String>();
     sList.add("hello");
     oList = sList;

     for (int i = 0; i < oList.size(); i++){
       System.out.println(oList.get(i));
     }

     // 编译失败
     // oList.add("123");
     #+END_EXAMPLE
     Java编译器通过限制方法参数列表中包含类型参数的方法调用，保证运行时的类型安全。也就是方法签名如 ~void test(T a)~ ，其中的 ~T~ 会被 ~? extends Object~ 替换掉，即使方法体并没有用到。

     *特点：*
     1) 禁止向泛型类执行写入操作
     2) 允许安全地从泛型类中读取数据
     
**** Java中的逆变
     #+BEGIN_EXAMPLE java
     List<? super Fruit> superFruitList = new LinkedList<Object>();
     superFruitList.add(new Orange());
     superFruitList.add(new Fruit());
     
     Object o = superFruitList.get(0);
     Fruit f = (Fruit)superFruitList.get(1);
     #+END_EXAMPLE

     *特点：*
     1) 允许安全地向泛型类执行写入操作
     2) 无法保证运行时从泛型类中读取数据类型的安全性

**** 为什么需要泛型的型变
     在泛型类型安全的基础上，提高API的灵活性。

    - ~K~ Key

    - ~T~ Type
    - ~V~ Value
    - ~S,U,V~ 第2,3,4个类型参数

** 方法的可变参数
   JDK1.5引入的特性，支持传递同类型的可变参数给方法，并且必须作为最后一个参数。
   #+BEGIN_EXAMPLE java
   public class Test{
     public void printMax(String str, double... numbers){
       // do something
     }

     public void execute(){
       String firstArg = "the first argument";
       double[] vals = {1d,2d,3d};
       this.printMax(firstArg, 1d,2d);
       this.printMax(firstArg, vals);
     }
   }
   #+END_EXAMPLE

*** 方法重载优先匹配固定参数
    若调用的方法能匹配到固定参数重载方法和可变参数重载方法，那么优先匹配固定参数重载方法。
    #+BEGIN_EXAMPLE java
    public class Test{
      public static String concat(String a, String b){
        System.out.println("concat(String a, String b)");
        return a + b;
      }
      public static String concat(String... strs){
        System.out.println("concat(String... strs)");
        StringBuilder sb = new StringBuilder();
        for (String str : strs){
          sb.append(str);
        }
        return sb.toString();
      }

      public static void main(String[] args){
        concat("a", "b"); // concat(String a, String b)
        concat("a", "b", "c"); // concat(String... strs)
      }
    }
    #+END_EXAMPLE

*** 如匹配到多个可变参数重载方法，则会报编译时异常
    #+BEGIN_EXAMPLE java
    public class Test{
      public static void test(String a, Integer... b){
        System.out.println("Integer");
      }
      public static void test(String a, String... b){
        System.out.println("String");
      }

      public static void main(String[] args){
        // test("abc", null); 由于无法判断null的数据类型，因此会抛出编译时异常
        String[] b = null;
        test("abc", b); // String
      }
    }    
    #+END_EXAMPLE

*** 反射
    由于反射是在运行时获取类、方法等的元数据，而可变参数在运行时被编译为数组。因此通过反射调用可变参数方法时，可变参数用数组代替。
    #+BEGIN_EXAMPLE java
    public class Test{
      public static void print(String... strs){
        System.out.println(strs[0]);
      }

      public static void main(String[] args){
        Method method = Test.class.getMethod("print", String[].class);
        method.invoke(null, new String[]{"a", "b"}); // 成功
        method.invoke(null, "a", "b"); // 抛出java.lang.IllegalArgumentException: wrong number of arguments
      }
    }
    #+END_EXAMPLE

** 内部类
   作用：
   1) 当要实现的接口方法与当前类的方法签名冲突时，可通过内部类实现接口方法(内部类可访问外部类所有成员)；
   2) 通过内部类继承其它类型，达到多继承的效果；
    
*** 分类
    1) 成员内部类
       #+BEGIN_EXAMPLE java
       public class Fruit {
         private double price = 100;
        
         private class InnerPrice implements Price{
           public double getPrice(){
             // discount 在Price接口中定义
             return price * discount;
           }
         }

         public class InnerTimeToMarket extends TTM {
           private double price = 200;

           @Override
           public Date getTTM(){
             // 当内部类和外部类存在同名成员变量/方法时，调用外部类成员变量/方法则要采用如下方式
             // 即内部类拥有指向外部类的引用
             System.out.printf("OuterClass price is: %d", Fruit.this.price);
             System.out.printf("InnerClass price is: %d", price);
             return new Date();
           }
         }

         public Price askPrice(){
           return new InnerPrice();
         }
       }
      
       // 调用
       Fruit f = new Fruit();
       // 通过private或protected隐藏内部类，调用则只需调用接口或父类约定的方法即可
       Price p = f.askPrice();
       p.getPrice();

       // 通过public暴露内部类，需要先实例化外部类，再实例化其内部类
       Fruit.InnerTimeToMarket ttm = f.new InnerTimeToMarket();
       #+END_EXAMPLE
       - 注意：
         1) 成员内部类不能拥有静态变量
         2) 成员内部类不能拥有静态方法
         3) 成员内部类不能拥有静态内部类
         4) 成员内部类可定义 ~static final~ 常量
    2) 静态内部类
       #+BEGIN_EXAMPLE java
       public static class Fruit {
         private static double price = 100;
        
         public static class InnerTimeToMarket extends TTM {
           private static double price = 200;

           @Override
           public Date getTTM(){
             // 当内部类和外部类存在同名静态变量/方法时，调用外部类静态变量/方法则要采用如下方式
             System.out.printf("InnerClass price is: %d", Fruit.price);
             System.out.printf("InnerClass price is: %d", price);
             return new Date();
           }
         }
       }

       // 调用
       Fruit.InnerTimeToMarket ttm = new Fruit.InnerTimeToMarket();
       Date time = ttm.getTTM();
       #+END_EXAMPLE
       - 注意：
         1) 静态内部类必须定义在静态类中
         2) 静态内部类中仅能访问外部类的静态变量和静态方法
    3) 局部内部类
       #+BEGIN_EXAMPLE java
       if (true){
         final String name = "john";
         class InnerClass{
           public void print(){
             System.out.println(name);
           }
         }
         InnerClass n = new InnerClass();
       }
       #+END_EXAMPLE
       - 注意
         1) 定义在成员方法的局部内部类，可以访问外部类的所有成员变量和成员方法
         2) 定义在静态方法的局部内部类，可以访问外部类的所有静态变量和静态方法
         3) 局部内部类可以访问方法中的修饰为 ~final~ 的局部变量和方法入参
            - 原理：
              由于局部变量随调用栈帧的释放而被回收，而内部类实例有可能依然可被访问，两者生命周期不一致。而实际上局部变量将作为构造方法入参，参与局部内部类的实例化过程。但存在如下问题，局部变量为基础类型时，Java默认采用传值方式传参。为避免开发人员误会在局部内部类操作的结果会影响到局部变量，因此通过 ~final~ 修饰局部变量，将基本类型的变量标记为常量，引用类型的则直接传递引用本身。
    4) 匿名内部类
       #+BEGIN_EXAMPLE java
       // new interfacename(){} 或 new superclassname(){}
       if (true){
         Fruit f = new Fruit(){
           public double getPrice(){
             return 1d;
           }
         }
       }
       #+END_EXAMPLE

* 异常处理
  1) 异常类型
     - ~java.lang.Throwable~ 所有异常的顶层父类
       - ~java.lang.Error~ 包括虚拟机错误和线程死锁，无法被程序员捕获处理，因此一旦抛出Error异常程序则会挂掉。
       - ~java.lang.Exception~ 可被获取和处理
         - ~java.lang.Runtime~ 运行时异常
         
  2) 异常分类
     1) unchecked exception(非检查性异常)
        Error类型异常、Runtime类型及其派生类异常。这类异常在编译时不被提发现和提示，开发人员应该通过修改代码的方式来杜绝这类异常，而不是依赖异常处理器处理(~try...catch~)处理。
        | 异常                            | 描述                                                                                                                 |
        |---------------------------------+----------------------------------------------------------------------------------------------------------------------|
        | ArithmeticException             | 当出现异常的运算条件时，抛出此异常。例如，一个整数"除以零"时，抛出此类的一个实例。                                   |
        | ArrayIndexOutOfBoundsException  | 用非法索引访问数组时抛出的异常。如果索引为负或大于等于数组大小，则该索引为非法索引。                                 |
        | ArrayStoreException             | 试图将错误类型的对象存储到一个对象数组时抛出的异常。                                                                 |
        | ClassCastException              | 当试图将对象强制转换为不是实例的子类时，抛出该异常。                                                                 |
        | IllegalArgumentException        | 抛出的异常表明向方法传递了一个不合法或不正确的参数。                                                                 |
        | IllegalMonitorStateException    | 抛出的异常表明某一线程已经试图等待对象的监视器，或者试图通知其他正在等待对象的监视器而本身没有指定监视器的线程。     |

** Java2后出现的集合框架
   集合框架的设计目标：
   1) 提供高性能的集合操作
   2) 对不同类型的集合采用类似的方式操作
   3) 对某个集合的扩展和适应必须简单
   
   组成：
   1) 接口
      1) Collection
         标识存储一组不唯一、无序的元素。
      2) List
         继承Collection接口，标识存储一组不唯一、有序的元素。
         - 特点
           1) 检索效率高
           2) 插入、删除效率低(会引起其它元素位置改变)
      3) Set
         继承Collection接口，标识存储一组唯一、无序的元素。
         - 特点
           1) 检索效率低
           2) 插入、删除效率高(不会引起其它元素位置改变)
      4) Queue
         继承Collection接口，标识一组遵循FIFO操作的元素。
      5) Map
         标识一系列键值对。
         1) ~Set<K> keySet([V defaultValue])~ 返回可变的键集合，若删除键集合的元素则会自动删除映射对应的条目，若向键集合添加元素，那么需要传入defaultValue作为值的默认值。
      6) SortedMap
         继承Map接口，标识一系列将key按升序排列的键值对。
      7) NavigableMap
         继承SortedMap接口。
      8) Map$Entry
         在Map接口内部定义的，标识一组键值对。
   2) 抽象类/实现类
      1) AbstractCollection
         实现大部分Collection接口的方法。
      2) AbstractList
         继承AbstractCollection和List，并实现大部分List接口的方法。
      3) AbstractSequentialList
         继承AbstractList，提供对集合元素的链式访问而不是随机访问。
      4) LinkedList
         链式数据结构的列表，非线性安全。
         - 特点
           1) 检索效率低
           2) 插入、删除效率高
         - 线性安全改造

           #+END_EXAMPLE
      5) ArrayList
         大小可变的数组，非线性安全。
         - 特点
           1) 检索效率高
           2) 插入、删除效率低
         - 线性安全改造
           #+BEGIN_EXAMPLE java
           List<String> unsafeList = new LinkedList();
           List<String> safeList = Collections.synchronizedList(unsafeList);
           #+END_EXAMPLE
      6) Vector
         继承AbstractList
      7) Stack
         继承Vector，提供一组遵循FILO操作集合
      8) AbstractSet
         继承AbstractCollection和Set，并实现大部分Set接口的方法。
      9) HashSet
         存储一组唯一、无序的元素。         
      10) LinkedHashSet
          具有可预知迭代顺序的Set接口的hash和linkedlist的实现
      11) TreeSet
          提供排序等功能
      12) AbstractMap
          集成Map接口，实现大部分Map的方法
      13) HashMap
          继承AbstractMap，根据键的HashCode值存储数据，检索效率高，非线性同步
      14) Hashtable
          继承AbstractMap，根据键的HashCode值存储数据，检索效率高，线性同步
      15) SortedMap
          继承Map接口，标识提供排序功能
      16) TreeMap

      17) WeakHashMap
          继承AbstractMap，使用弱密钥的哈希表
      18) LinkedHashMap
          继承HashMap，对键按自然顺序来排序
      19) IdentityHashMap

  System.out.println(int[].class.getCanonicalName());    // 显示 int[]

  Student student = new Student();
  student.getClass().getName(); // 返回com.john.Student

  Class<?> cObj1 = Student.class;

  Class<?> superClass = cObj1.getSuperclass();

  Constructor<?>[] cs = cObj1.getConstructors();

  Method[] fs = cObj1.getDeclaredMethods();

  Method[] fs1 = cObj1.getMethods();
  for (Method f : fs1){

    String functionName = f.getName();
    // 获取方法修饰符
    Modifier<?>[] ms = f.getModifiers();
    String modifierNames = Modifier.toString(ms); // 方法所有修饰符组成的字符串
    // 获取方法入参列表类型
    Class<?>[] parameterTypes = f.getParameterTypes();
    // 获取方法声明抛出的异常
    Class<?>[] exceptionTypes = f.getExceptionTypes();
    // 获取方法定义所在的类型
    Class<?> declaringClass = f.getDeclaringClass();
  }

  Field[] fields = cObj1.getDeclaredFields();

  Field[] fields1 = cObj1.getFields();

* 从classpath获取资源
  ClassLoader是从classpath中读取资源的类，一般情况通过ClassLoader加载字节码文件(.class文件)，但它提供两个方法加载其他资源。
  - ~public URL getResource(String name)~
    实际代码中一般是通过 ~YourClass.class.getResource(String name)~ 来调用，通过委托底层依然是调用ClassLoader的该方法。
  - ~public InputStram getResourceAsStream(String name)~
    实际代码中一般是通过 ~YourClass.class.getResourceAsStream(String name)~ 来调用，通过委托底层依然是调用ClassLoader的该方法。
  - ~public static URL getSystemResource(String name)~ 静态方法调用该方法获取classpath下的资源。
  - ~public static InputStream getSystemResourceAsStream(String name)~ 静态方法调用该方法获取classpath下的资源。

* ClassLoader(类加载器)
  类文件可以存放在文件系统、JAR文件里、远程某个位置或者在内存中动态构建。而类加载器就是负责加载字节流，并且在虚拟机中将它们转化为一个类或接口。

  1) Bootstrap ClassLoader
     C++实现属于虚拟机的一部分，作为Ext ClassLoader的父类加载器，负责jdk_home/lib目录下的核心API(如jre/lib/rt.jar)或-Xbootclasspath指定的jar包装入工作。
     - 代码中无法获取Bootstrap ClassLoader实例 ~String.class.getClassLoader() == null~
  2) System ClassLoader(系统类加载器)
     负责 ~java -classpath/-Djava.class.path~ 所指目录下的类和jar包装入工作。
     - 在Oracle Java中, ~this.getClass().getClassLoader() instanceof URLClassLoader~
  3) Ext ClassLoader(扩展类加载器)
     作为App ClassLoader的父类加载器，负责jdk_home/lib/ext目录下的jar包或-Djava.ext.dirs指定目录下的jar包等"标准库扩展"装入工作。
     - 在Oracle Java中, ~this.getClass().getClassLoader() instanceof URLClassLoader~
  4) App ClassLoader
     负责加载用户类。
  5) Customer ClassLoader
     一般都继承App ClassLoader，用户自定义。

** 类加载器反转(classloader inversion)
   实现动态加载类的方法所用的类加载器，与加载其自身的类加载器不同，称为类加载器反转。
   - 问题示例
     #+BEGIN_EXAMPLE java
     package com.john;
  
     class T {
       public void load(String name){
         Class.forName(name);
       }
     }
  
     class Main {
       public void plugin(){
         URL[] urls = {
           new URL("file:///path/to/directory")
         };
         URLClassLoader loader = new URLClassLoader(urls);
         Class<?> tClass = Class.forName("com.john.T", true, loader);
         tClass.getMethod("load").invoke(tClass.newInstance(), "com.john.TT");
       }
     }
     #+END_EXAMPLE
   - 解决办法——上下文类加载器
     #+BEGIN_EXAMPLE java
     package com.john;
  
     class T {
       public void load(String name){
         Class.forName(name, true, Thread.currentThread().getContextClassLoader());
       }
     }
  
     class Main {
       public void plugin(){
         URL[] urls = {
           new URL("file:///path/to/directory")
         };
         URLClassLoader loader = new URLClassLoader(urls);
         ClassLoader cl = Thread.currentThread().getContextClassLoader();
         Thread.currentThread().setContextClassLoader(loader);
         Class<?> tClass = Class.forName("com.john.T", true, loader);
         tClass.getMethod("load").invoke(tClass.newInstance(), "com.john.TT");
         Thread.currentThread().setContextClassLoader(cl);
       }
     }
     #+END_EXAMPLE

* JNDI(Java Naming and Directory Interface)
  是一组在Java中访问命名和目录服务的API，作为J2EE的核心技术之一。
  1) 命名服务
     将名称和对象联系起来，让我们可以通过名称访问对象。
  2) 目录服务
     基于命名服务上作增强，使得对象不仅有名称还有属性。
  
  JNDI组成
  1) 应用编程接口(API)
  2) Naming Manager
  3) 服务提供者接口(SPI,Service Provider Interface)
  4) LDAP服务提供者、NIS服务提供者、文件系统服务提供者等

  优点：通过名称引用Java对象或资源，而不必知道对象或资源的物理位置。
   
  JNDI现在可访问的命名和目录服务有
  1) DNS
  2) XNam
  3) Novell目录服务
  4) LDAP
  5) CORBA对象服务
  6) 文件系统
  7) Windows的注册表
  8) RMI
  9) DSML v1&v2
  10) NIS
  
  在J2EE环境下，JNDI的主要用途
  1) 获得Java事务API中的UserTransaction接口的引用
  2) 获得如JDBC、JMS等连接到资源工厂
  3) 在EJB组件中，查找其他EJB组件

  JNDI的概念
  1) Atomic Name(原子名)
     不可分割的名字，如/ETC/FSTAB中的ETC和FSTAB均为Atomic Name
  2) Compound Name(复合名)
     由0或多个Atomic Name组成，如/ETC/FSTAB就是Compound Name
  3) Binding(绑定)
     名字和对象或资源的关联
  4) Context(上下文)
     包含0或多个Binding，如java/MySql中java为Context，而MySql为名字，整体和某个对象或资源关联起来了
  5) Subcontext(子上下文)
     上下文的上下文，如java/ejb/helloBean中ejb为Subcontext
  6) 命名系统
     构成命名空间定义的语法语义
  7) 命名空间
     根据命名系统的语法约定由原子名、复合名、上下文构成的结构，如文件系统的命名空间由目录和文件名组成
  8) 合成名
     由多个命名系统组成的名字，如URL ~http://java.sun.com/product/index.html~
     1) http来自URL scheme-id命名系统
     2) java.sum.com来自DNS命名系统
     3) product、index.html来自文件系统的命名系统

** Lambda表达式
   由编译器提供支持。
   lambda表示函数参数，而带有参数变量的表达式被称为lambda表达式。
   而Lambda表达式在Java中为实现特定 *接口或抽象类* 的 *抽象方法* 的便捷语法。

   #+BEGIN_EXAMPLE java
   Comparator<String> comparator = (String a, String b) -> Integer.compare(a.length(), b.length());
   // 对于函数式接口，方法类型可被推导，因此参数列表可以省略入参类型
   Comparator<String> comparator = (a, b) -> Integer.compare(a.length(), b.length());
   Comparator<String> comparator = (a, b) -> {
     int aLen = a.length();
     int bLen = b.length();

     return Integer.compare(aLen + 1, bLen);
   }
   
   // 参数列表为空时
   Runnable task = () -> {
     for (int i = 0; i < 1000; i++){
       // do something
     }
   };

   // 仅有一个参数时，可以省略括号
   Predicate<String> predicate = x -> Objects.isNull(x);
   #+END_EXAMPLE

*** 作用域
    Lambda表达式的方法体和所在代码块有着相同的作用域。
    1) Lambda表达式中不能有与局部变量同名的参数或局部变量
    2) this指向所在代码块所属的实例
    3) 只能访问声明为final的自由变量
       - 什么是自由变量：不是从入参传递过来的参数
         #+BEGIN_EXAMPLE java
         final int a = 1;
         Runnable task = () -> System.out.println(a); // a就是自由变量
         #+END_EXAMPLE
       - 什么是闭包(closure)：含自由变量的代码块
       - 增强for循环的变量自动实现final
         #+BEGIN_EXAMPLE java
         for (String arg : args){
           new Thread(() -> System.out.println(arg)).start();
         }
         #+END_EXAMPLE

        1) ~类::成员方法~
        2) ~类::静态方法~
        3) ~实例::成员方法~
      - 对于重载方法
        编译器会根据调用上下文自动编译为合适的重载方法
      - 示例
        #+BEGIN_EXAMPLE java
        // 编译为Arrays.sort(new String[]{"abc", "Abc"}, (a, b) -> a.compareToIgnoreCase(b));
        Arrays.sort(new String[]{"abc", "Abc"}, String::compareToIgnoreCase);

        // 编译为 Comparator<String> comparator = (String a, String b) -> Integer.compare(a.length(), b.length());
        Comparator<String> comparator = Integer::compare;

        // 编译为 list.forEach((String x) -> System.out.println(x));
        list.forEach(System.out::println);

        // 内部类调用外部类实例的成员方法引用
        Outer.this::method

        // 调用父类实例的成员方法引用
        super::method
        #+END_EXAMPLE
   2) 构造方法引用
      1) 对象
         - 语法： ~类::new~
         - 示例
           #+BEGIN_EXAMPLE java
           List<Role> roles = names.stream().map(Role::new);
           #+END_EXAMPLE
      2) 数组
         - 语法： ~类[]::new~
         - 示例
           #+BEGIN_EXAMPLE java
           // Role[]::new 编译为 n -> new Role[n]
           Role[] roles = stream.toArray(Role[]::new);
           #+END_EXAMPLE

** Stream API
   简单示例
   #+BEGIN_EXAMPLE java
   // 获取文本中长度大于12的单词数

   String content = new String(Files.readAllBytes(Paths.get("alice.txt")), StandardCharsets.UTF_8);
   // 非字母作为分隔符，拆分单词
   List<String> words = Arrays.asList(content.split("\PL+"));
   
   long count = words
       .stream()                            // 创建初始流
       .filter(word -> word.length() > 12)  // 将初始流转换为其他流程的中间操作
       .count();                            // 执行终止操作产生结果，该操作会触发懒惰操作执行，此后前面的流将不能再使用
   #+END_EXAMPLE

*** 创建流
    1) 集合及其派生类型(Set,List,Queue,Stack)转换为Stream实例： 通过接口 ~Collection~ 提供的stream方法
    2) 数组转换为Stream实例：
       1) ~Stream<T> Stream.of(T[])~
       2) ~Stream<T> Stream.of(T...)~
       3) ~Stream<T> Arrays.stream(T[] array, int from, int end)~
    3) 不包含元素的流： ~Stream<String> silence = Stream.empty()~
    4) 无限流
       1) ~Stream<T> Stream.generate(Supplier<T> supplier)~
          #+BEGIN_EXAMPLE java
          Stream<Double> randoms = Stream.generate(Math::random);
          #+END_EXAMPLE
       2) ~Stream<T> Stream.iterate(T init, UnaryOperator<T> unaryOperator)~
          #+BEGIN_EXAMPLE java
          Stream<BigInteger> integers = Stream.iterate(BigInteger.ZERO, i -> i.add(BigInteger.ONE));
          #+END_EXAMPLE
    5) 其它帮助方法
       1) ~Stream<String> Pattern.compile("\\PL+").splitAsStream(String content)~
       2) ~Stream<String> Files.lines(String path)~ 返回所有行的流

*** 流转换
    1) ~Stream<T> filter(Predicate<T> pred)~ 过滤元素构造新的流
    2) ~Stream<R> map(Function<T, R> function)~ 加工元素构造新的流
    3) ~Stream<R> flatMap(Function<T, Stream<R>> function)~ 每个元素生成新的流，然后将素有流合并成为新的流
 
*** 提取子流和组合流
    1) ~Stream<?> limit(int n)~ 返回包含n个元素的流
    2) ~Stream<?> skip(int n)~ 返回丢弃前n个元素的流
    3) ~Stream<T> Stream.concat(Stream<T> a, Stream<T> b)~ 合并两个流。注意：第一个流不能为无限流，否则无法合并。

*** 其他转换
    1) ~Stream<?> distinct()~ 返回一个元素顺序相同当消除重复元素的流
    2) ~Stream<?> sorted()~ 

*** 归约(reduction)
    归约作为终止操作，可以将流归为一个可以在程序中使用的非流值。
    1) ~int count()~ 返回流中的元素个数
    2) ~Optional<?> max(Comparable<?> comparable)~ 返回最大值
    3) ~Optional<?> min(Comparable<?> comparable)~ 返回最小值
    4) ~Optional<?> findFirst()~ 返回第一个元素
    5) ~Optional<?> findAny()~ 返回任意一个元素，在并行计算中十分有用
    6) ~boolean anyMatch(Predicate<?> predicate)~ 检查流中是否含有某个匹配的元素
    7) ~boolean allMatch(Predicate<?> predicate)~ 检查流中是否所有元素都匹配

*** 收集器
    1) ~Iterator<T> iterate()~ 将Stream转换为Iterator实例
    2) ~void forEach(Consumer<T> consumer)~
    3) ~T[] toArray(T[] array)~ 将Stream转换为数组
       #+BEGIN_EXAMPLE java
       String[] elements = stream.toArray(String[]::new);
       #+END_EXAMPLE
    4) ~List<T> collect(Collections.toList())~
    5) ~Set<T> collect(Collections.toSet())~
    6) ~TreeSet<T> collect(Collections.toCollection(TreeSet::new))~
    7) ~String collect(Collections.joining())~ 将流中所有字符拼接为字符串
    8) ~String collect(Collections.joining(","))~ 将流中所有字符拼接为字符串，并指定字符间的分隔符
    9) ~Int/Long/DoubleSummaryStatistics Collections.summarizingInt/Long/Double(Function<T, R> function)~ 返回归约为总数、平均值、最大值和最小值
       #+BEGIN_EXAMPLE java
       IntSummaryStatistics summary = stream.collect(
       Collections.summarizingInt(String::length
       ));
       summary.getAverage();
       summary.getMax();
       summary.getMin();
       #+END_EXAMPLE
    10) ~Collections.toMap~ 将Stream转换为Map实例
        #+BEGIN_EXAMPLE java
        // 从Stream元素中提取名字和年龄作为键值对，当出现重复键值时抛出IllegalStateException
        Map<String, Integer> idToAge = stream.collect(
          Collections.toMap(Person::getName, Person::getAge));
        
        // 从Stream元素中提取名字和元素自身作为键值对，当出现重复键值时抛出IllegalStateException
        Map<String, Integer> idToEntity = stream.collect(
          Collections.toMap(Person::getName, Function.identity()));

        // 第三个参数在出现重复键值时被调用，用于决定如何处理
        Stream<Locale> locales = Stream.of(Locale.getAvailableLocales());
        Map<String> languageNames = locales.collect(
          Collections.toMap(Locale::getDisplayName, Locale::getDisplayName, (existingValue, newValue) -> existingValue));
        
        // 汇总重复键的值
        Map<String> languageNames = locales.collect(
          Collections.toMap(Locale::getDisplayCountry
                            , x -> Collection.singleton(x.getDisplayName())
                            , (existingValue, newValue) -> {
                              Set<String> union = new HashSet<String>(existingValue);
                              union.addAll(newValue);
                              return union;
                            }));
                  
        // 指定Map构造方法
        Map<String> languageNames = locales.collect(
          Collections.toMap(Locale::getDisplayName
                            , Locale::getDisplayName
                            , (existingValue, newValue) -> existingValue
                            , TreeMap::new));
        #+END_EXAMPLE
    11) ~Collection.toConcurrentMap~ 和 ~Collection.toMap~ 用于一致但会生成并发Map
    
*** 分组和分片
    1) 分组 ~Collections.groupingBy(Function<T, R> function)~ 或 ~Collections.groupingByConcurrent(Function<T, R> function)~
       根据某个条件将流分为多个组
       #+BEGIN_EXAMPLE java
       Map<String, List<Locale>> countryToLocales = locales.collect(
         Collections.groupingBy(Locale::getCountry));
       #+END_EXAMPLE
       - 下游收集器(Downstream)
         上文的收集器均可以用作下游收集器，即对分组内容进行运算得到新值
         1) ~Collections.toSet()~ 将List转换为Set
         2) ~Collections.counting()~ 统计个数
         3) ~Collections.summingInt|Long|Double~ 求和
         4) ~Collections.maxBy(Comparator)~ 求最大值
         5) ~Collections.minBy(Comparator)~ 求最小值
         6) ~Collections.mapping(Function<T, R> function, 下游收集器)~ 对分组中元素调用function进入运算，然后再通过子下游收集器处理
    2) 分片 ~Collections.partitioningBy(Predicate<T> pred)~ 或 ~Collections.partitioningByConcurrent(Function<T, R> function)~
       根据某个条件将流分为两个组
       #+BEGIN_EXAMPLE java
       Map<Boolean, List<Locale>> enAndOtherLocales = locales.collect(
         Collections.partitioningBy(l -> l.getLanguage().equals("en"));       
       Lis<Locale> englsh = enAndOtherLocales.get(true);
       #+END_EXAMPLE

*** 归约操作(reduce)
    #+BEGIN_EXAMPLE java
    // 流中第一个元素作为累计初始值 reducing(T accu, T item)
    stream.reduce(BinaryOperator<T> reducing)

    // 指定一个值作为累计初始值 reducing(T accu, T item)
    stream.reduce(T init, BinaryOperator<T> reducing)

    // 并行流中通过最后一个方法将多个归约合并
    stream.reduce(T init, BinaryOperator<T> reducing, BinaryOperator<T> sum)
    #+END_EXAMPLE

*** 基本数据的流
    提供IntStream、LongStream、DoubleStream和OptionalInt等针对基本数据类型的流，减少装箱拆箱的性能损耗。

*** 并行流
    在执行终止操作前都可以将串行流修改为并行流。
    注意：在执行终止操作后，不能修改流的底层数据结构的元素

**** 创建
     #+BEGIN_EXAMPLE java
     List<String> words = new ArrayList<String>();
     String[] wordArray = words.toArray(String[]:new);
     Stream<String> pWords1 = words.parallelStream();
     Stream<String> pWords2 = Stream.of(wordArray).parallel();
     #+END_EXAMPLE

**** 必须保持操作无状态
     #+BEGIN_EXAMPLE java
     Map<Integer, Long> shortWordCounts = pWords1
       .filter(s -> s.length() < 10)
       .collect(groupingBy(String::length, counting()));
     #+END_EXAMPLE

**** 无序操作效率更高
     默认情况下并行Stream会根据流中元素的顺序，按序输出结果序列。但如distinct、counting和limit等操作在无序下效率更高。而groupingBy在有序情况下效率远低于无序。
     #+BEGIN_EXAMPLE java
     Stream<String> sample =pWords1.unordered().limit(3);

     // 返回值为无序
     Map<Integer, Long> sample2 = pWords1.collect(groupingByConcurrent(String::length));
     #+END_EXAMPLE

** Optional类
   在非完全测试的程序中返回null，会导致空指针异常，因此Optional类是一种更好的表明缺少返回值的方式。
   
*** 使用
    #+BEGIN_EXAMPLE java
    Optinal<String> os = Stream.of(new String[]{"123"}).filter(x -> x == "321").findFirst();

    os.orElse("default value");
    // 若没有值则返回默认值
    os.orElseGet(() -> System.getProperty("user.value"));
    // 若没有值则抛出异常
    os.orElseThrow(IllegalStateException::new);
    // 若存在值则执行方法
    os.ifPresent(Comsumer<?> comsumer);
    // 若存在值则执行方法
    Optional<R> result = os.map(Function<T, R> predicate);
    // 如存在值则返回，否则抛出NoSuchElementException
    os.get();
    // 若存在值则返回true，否则返回false
    os.isPresent();
    #+END_EXAMPLE

* [[https://www.projectlombok.org/features/all][Lombok]]
  通过注解在编译时自动附加代码，从而简化Java代码和加速开发效率。

** 使用  
   1) ~@Data~
      - 可用于：类
      - 作用：为类的所有属性生成setter/getter(若属性为final则不生成setter)、equals、canEqual、hashCode、toString方法
      - 示例
        #+BEGIN_EXAMPLE java
        import lombok.Data;

        @Data
        public class Test{
          private final String a;
          private String b;
        }
        // 编译输出
        public class Test {
          private final String a;
          private String b;
         
          public String getA(){ return a;}
          public String getB(){ return b;}
          public void setB(String b){ return this.b = b;}
         
          @Override
          public boolean equals(Object o){
            if (o == this) rerturn true;
            if (!(o instanceof Test)) return false;
            Test other = (Test)o;
            if (!other.canEqual(this)) return false;
            if (this.getA() == null ? other.getA() != null : !this.getA().equals(other.getA())) return false;
            if (this.getB() == null ? other.getB() != null : !this.getB().equals(other.getB())) return false;
            return true;
          }
         
          @Override
          public int hashCode(){
            final int PRIME = 59;
            int result = 1;
            final long temp1 = Double.doubleToLongBits(this.getScore());
            result = (result*PRIME) + (this.getName() == null ? 43 : this.getName().hashCode());
            result = (result*PRIME) + this.getAge();
            result = (result*PRIME) + (int)(temp1 ^ (temp1 >>> 32));
            result = (result*PRIME) + Arrays.deepHashCode(this.getTags());
            return result;
          }

          @Override 
          public String toString() {
            return "Test(" + this.getA() + ", " + this.getB() + ")";
          }

          protected boolean canEqual(Object other) {
            return other instanceof Test;
          }
        }
        #+END_EXAMPLE
   2) ~@Getter/@Setter~
      - 可用于：类/字段
      - 作用：自动生成setter/getter，还可以控制setter/getter的访问控制符
      - 示例：
        #+BEGIN_EXAMPLE java
        public class Test{
          @Getter
          @Setter
          private int age = 10;
         
          @Setter(AccessLevel.PROTECTED)
          private String name;
        }
        #+END_EXAMPLE
   3) ~@NonNull~
      - 可用于：方法参数
      - 作用：生成非空声明，用于校验参数，避免空指针
      - 示例
        #+BEGIN_EXAMPLE java
        public class Test {
          public Test(@NonNull String a){

          }
        }
        // 编译后

          }
        }

   4) ~@ToString~
      - 可用于：类

** 工作原理分析
   JDK5后注解提供两种解析方式：
   1) 运行时解析
      ~@Retention(RUNTIME)~ 标记注解为运行时解析，通过反射获取注解信息(~java.lang.reflect.AnnotatedElement~ 接口)
   2) 编译时解析
      - 两种机制
        1) Annotation Processing Tool
           JDK5引入但JDK7废弃，JDK8直接被删除。被替换的原因：1. API都在com.sum.mirror非标准包中；2. 没有集成到javac中，需要额外运行。
        2) Pluggable Annotation Processing API([[https://jcp.org/en/jsr/detail?id=269][JSR 269]])
           JDK6引入用于替代Annotation Processing Tool.
           - 工作流程
             1) javac对源码进行分析，生活了一棵抽象语法树(AST)
             2) 运行中调用实现了JSR 269 API的Lombok程序
             3) Lombok程序对AST进行处理，根据注解生成树节点
             4) javac根据AST生成字节码文件
