* 基础概念
1) JDK和JRE的区别
   JRE(Java Runtime Environment)仅包含Java程序运行时必要的虚拟机和核心库组件；而JDK(Java Development Kit)除了包含JRE外，还包含一系列开发、诊断工具(如jps等)。
2) J2SE、J2EE和J2ME的区别
   1998年JDK1.2版本发布时，将Java技术体系拆分为J2SE、J2EE和J2ME三大技术体系。
   - J2SE(Java 2 Standard Edition)，包含Java语言核心库(如数据库连接、网络编程、接口定义等)，主要用于桌面应用软件的编程；
   - J2EE(Java 2 Enterprise Edition)，除了包含J2SE的内容，还包含如Servlet、JSP、EJB等组件，主要用于分布式网络程序的开发；
   - J2ME(Java 2 Micro Edition)，只包含J2SE的一部分，主要用于手机、PDA等消费类电子产品软件开发。
3) J2SE和Java SE的区别
   JDK1.6发布时更名为JDK6，同时J2SE更名为Java SE 6。

  - byte, 1byte signed integer
    - default 0
    - min -128
    - max 127
    - 包装类 Byte
  - short, 2byte signed integer
    - default 0
    - min -32768
    - max 32767
    - 包装类 Short
  - int, 4byte signed integer
    - default 0

    - default 0.0f

    - 包装类 Float
  - double, 8byte IEEE 754 float
    - default 0.0d
    - 不能用于表示精确的值，如货币
    - 包装类 Double
  - boolean
    - default false
    - 包装类 Boolean
  - char, a 2byte Unicode character
    - default \u0000
    - min \u0000
    - max \uffff
    - 包装类 Character

* 局部变量、实例变量和静态变量
  1) 局部变量

     - 没有默认值，因此声明后必须经过初始化后才能使用
  2) 实例变量

       2) 构造方法中设置
  3) 静态变量
     - 在静态存储区分配存储空间
     - 有默认值
       - 数值型变量为0

       - 引用类型变量为null
     - 变量值的初始化时机
       1) 声明时设置
       2) 构造方法中设置
       3) 静态语句块中设置

* 日期时间

** ~java.util.Date~
   - 构造方法
     1) ~new Date()~ 使用当前日期时间初始化对象
     2) ~new Date(long millisec)~ 参数作为相对于1970/1/1起的毫秒数来初始化对象
   - 实例方法
     很多方法已经被标识为deprecated，建议使用Calender来操作
     - ~boolean after(Date date)~ 判断Date实例是否在入参之后
     - ~boolean before(Date date)~ 判断Date实例是否在入参之前
     - ~Object clone()~ 复制实例副本
     - ~boolean equals(Date date)~ Date实例和入参相同则返回true
     - ~int hashCode()~ 返回对象的哈希码
     - ~int compareTo(Date date)~ Date实例与入参相同则返回0；Date实例在入参之前则返回负数；Date实例在入参之后则返回正数
     - ~long getTime()~ 返回自1970/1/1 00:00:00 GMT以来的毫秒数
     - ~void setTime(long time)~ 参数作为相对于1970/1/1起的毫秒数来设置日期时间对象值
     - ~String toString()~ 返回格式为 ~dow mon dd hh:mm:ss UTC yyyy~ 格式的日期时间字符串，其中dow为Sun,Mon,Tue,Web,Thu,Fri,Sat

*** 文件操作

FileInputStream每次调用read都会从磁盘读取一个字节到内存，可以 ~new BufferedInputStream(new FileInputStream("/path/to/file"))~ 从磁盘一次读取一大块数据到内存，然后再逐个字节读出来处理。
FileOutputStream每次调用write都会往磁盘写入一个字节的数据，可以通过 ~new BufferedOutputStream(new FileOutputStream("/path/to/file"))~ 往磁盘一次写入一大块数据。

*** 管道操作
管道可作为两个线程间字节、字符通信方式，一个管道必须由一个Input和一个Output关联出现，并且一个线程持有Input，另一个线程持有Output，Input和Output不能同一个线程持有。(read、write方法均为同步阻塞方法)

#+begin_example java
final PipedOutputStream output = new PipedOutputStream();
final PipedInputStream input = new PipedInputStream(output);

Thread t1 = new Thread(new Runnable(){
    @Override
    public void run(){
        try {
            output.write("Hi".getBytes());
        }
        catch(IOException e){}
    }
});

Thread t2 = new Thread(new Runnable(){
    @Override
    public void run(){
        try {
            int data = input.read();
            while (data != -1) {
                System.out.print((char) data);
                data = input.read();
            }
        }
        catch(IOException e){}
    }
});
t1.start();
t2.start();
#+end_example

* 修饰符
用于定义类、方法和变量，通常放在语句的最前端

** 访问控制修饰符
1) default 在同一个包内可见。使用对象：类、接口、方法、变量
2) public 公开。使用对象：类、接口、方法、变量
3) protected 对同一包内的类和所有子类可见。使用对象：方法、变量、内部类
4) private 在同一个类内可见。使用对象：方法、变量、内部类

*** 接口的默认访问修饰符
1) 变量默认为 ~public static final~
2) 方法默认为 ~public abstract~

** 非访问控制修饰符
1) final 用来修饰类、方法和变量
   1) 修饰类，类为最终类不能被继承
      1) 其下方法自动添加final修饰符
      2) 其下成员变量不会自动添加final修饰符
   2) 修饰方法，方法不能被子类重写(Override)
   3) 修饰变量，变量值不能被修改
      1) 变量为引用类型，那么变量指向的引用地址不能修改，但引用对应的内部属性可以被修改
         #+BEGIN_EXAMPLE java
         final Map<String, String> map = new HashMap<String, String>();
         map.put("one", "1");
         #+END_EXAMPLE
      2) 搭配static声明类变量作为常量使用
2) abstract 用来修饰抽象类和抽象方法
   1) 抽象类不能同时声明为final
   2) 抽象方法不能同时声明为final或static
   3) 若一个类包含抽象方法，那么该类也必须声明为抽象类
3) static 用来修饰类方法和类变量
4) synchronized 声明方法同一时间只能被一个线程访问
   - 示例 ~public synchronized void showDetails() {}~
5) volatile 修饰成员变量在每次线程访问时均强制从共享内存中重新读取值，并且当发生变化时强制线程将变化值回写到共享内存
6) transient 修饰变量不参与类的序列化

** 接口
   实现接口的抽象类可不实现接口的所有方法，否则则需要实现接口的所有方法。
   1) 接口和抽象类的区别
      1) 接口不能包含静态代码块和静态方法
      2) 一个类可实现多个接口，但只能继承一个抽象类
      3) 接口的方法不能有方法体，而抽象类中的非抽象方法将包含方法体
      4) 接口的成员变量修饰符只能为 ~public static final~ ，抽象类的成员变量修饰符和普通实现类一样
      5) 接口的成员方法修饰符只能为 ~public~
   2) 接口定义
      #+BEGIN_EXAMPLE java
      [public|private] interface <接口名称> [extends <其它接口名称>...]{}
      #+END_EXAMPLE
   3) 接口多继承
      接口支持多继承，即可通过 ~extends~ 继承多个父接口
      #+BEGIN_EXAMPLE java
      interface A{}
      interface B{}
      interface C extends A, B {}
      #+END_EXAMPLE
   4) 标记接口
      不包含任何属性和方法，仅用于表明它的实现类属于一个特定的类型拥有某个或某些权利。
   5) 接口可以拥有静态方法，一般作为工厂方法
   6) 接口可以拥有默认方法(Java8+)
      #+BEGIN_EXAMPLE java
      public interface Test{
        // 实现类可选择性地重写或沿用默认方法
        default void print(){
          System.out.println("test");
        }
      }
      #+END_EXAMPLE
      - 多接口方法冲突：若继承的多个接口定义了相同方法签名的方法
        - 若没有提供默认方法，那么派生类只需定义方法体，或定义为抽象类即可
        - 若存在默认方法，则派生类必须重写该方法
          #+BEGIN_EXAMPLE java
          interface A {
            default void print(){
              System.out.println("A");
            }
          }
          interface B {
            void print();
          }
          class C implements A, B {
            @Override
            public void print(){
              A.super.print(); // 调用接口的默认方法
            }
          }
          #+END_EXAMPLE

** 重写(Override)和重载(Overload)
   - 重写(Override)
     子类重新实现父类的签名相同的实例方法
     1) 子类方法不能抛出比父类方法范围更大的异常，仅能抛出父类方法声明的异常或其派生类
        #+BEGIN_EXAMPLE java
        class Super1{
          void echo throws IOException () {}
        }
        class Child1 extends Super1{
          // 编译时报错，因为子类方法抛出的异常范围比父类的更大
          void echo throws Exception () {}
        }
        #+END_EXAMPLE
     2) 子类方法的参数列表必须和父类的一致
     3) 子类方法的返回值类型要么和父类的一致，要么为父类方法返回值类型的派生类(Java5前必须和父类的一致)
     4) 子类方法的访问权限不能低于父类的
        1) 父类为public，那么子类必须为public
        2) 父类为default，那么子类可以为default或public
        3) 父类为protected，那么子类可以为protected、default或public
     5) 在子类方法中可通过 ~super~ 调用父类中被重写的方法
     6) 静态方法不存在重写，父类和子类的静态方法各自独立
   - 重载(Overload)
     在同一个类中多个名称相同但方法参数列表不同的实例或静态方法，重载方法的返回值类型可相同也可以不同
     1) 重载方法必须具有不同的参数列表，仅返回值类型不同是无法区分方法的

*** 底层原理
    类型擦写(type erasure)：编译器在编译时会将所有类型参数信息擦除掉，在运行时不存在任何类型参数相关的信息。如 ~List<String>~ 在运行时只剩下 ~List~ ，这样就能兼容Java5之前版本的二进制类库，因此称Java的泛型为伪泛型。
    由于类型参数信息对JVM来说是不可见的，因此无法避免在运行时出现的类型转换异常的情况，但也不会出现真泛型中的类型膨胀问题。

    问题示例1——运行时Class类实例相相等
    #+BEGIN_EXAMPLE java
    ArrayList<String> list1 = new ArrayList<String>();
    list1.add("abc");
    ArrayList<Integer> list2 = new ArrayList<Integer>();
    list2.add(123);

    // 返回true
    System.out.println(list1.getClass() == list2.getClass());
    #+END_EXAMPLE

    问题示例2——运行时反射方法调用
    #+BEGIN_EXAMPLE java
    ArrayList<String> list1 = new ArrayList<String>();
    list1.add("abc");
    // list1.add(123); 编译失败
    Method m = list1.getClass().getMethod("add", Object.class);
    m.invoke(list1, 123); // 运行正常
    #+END_EXAMPLE

**** 类型擦写原理
     编译时将所有类型参数(parameterized type)中指定的类型转换为原始类型(raw type)，然后自动在取值位置添加类型转换代码。
     1) 原始类型，即最后在字节码中类型变量真正的数据类型；
     2) 转换原始类型的规则
        1) 没有类型限定符时，统一转换为Object；
        2) 附带 ~extends~ 类型限定符，统一转换为第一个基类或接口；
           #+BEGIN_EXAMPLE java
           public <T extends Comparable & Serializable> T get(){ return null; }
           // 转换为public Comparable get() { return null; }
           #+END_EXAMPLE
           *建议：将标记接口放在限制列表的最后*
        3) 附带 ~super~ 类型限定符，统一转换为Object。
     3) 自动添加类型转换代码
        #+BEGIN_EXAMPLE java
        public <T> T get(T a){ return a; }
        Integer a = <Integer>get(1);

        // 编译后的代码逻辑如下
        public Object get(Object a){ return a; }
        Object oA = get(1);
        Integer a = (Integer)oA;
        #+END_EXAMPLE

**** 限制分析
     1) 类型变量不能为8种基础类型和void
        因为类型擦写后的原始类型为Object，基础类型强转Object无效，因此必须使用它们的封装类才行。
     2) 编译期仅根据引用类型变量的类型参数作检查，而不管实际引用的对象的类型参数
        #+BEGIN_EXAMPLE java
        ArrayList list1 = new ArrayList<String>();
        list1.add("abc");
        list1.add(123);

        ArrayList<String> list2 = new ArrayList();
        list2.add("abc");
        list2.add(123); // 编译失败
        #+END_EXAMPLE
     3) 静态方法和属性不能引用类的类型参数
        由于类的类型参数是在类实例化时才能确定，若静态方法和属性引用类的类型参数，则无法确认类型参数的实际类型
        #+BEGIN_EXAMPLE java
        public class Basket<E>{
          public static void add(E e); // 编译失败
        }
        #+END_EXAMPLE
     4) 不支持类型参数的直接实例化，但可以通过反射实现
        #+BEGIN_EXAMPLE java
        public <T extends Comparable> T[] test(T[] a){
          // T m = new T(); 编译失败
          // T[] ms = new T[2]; 编译失败

          T[] ms = (T[])Array.newInstance(a.getClass().getComponentType(), 2);
          return ms;
        }
        #+END_EXAMPLE
     5) 重写含类型参数的方法
        #+BEGIN_EXAMPLE java
        class P<T>{
          private T val;
          public void set(T val){
            this.val = val;
          }
          public T get(){
            return this.val;
          }
        }

        class C extends P<Date>{
          @Override
          public void set(Date val){
            super.set(val);
          }
          @Override
          public Date get(){
            return super.get();
          }
        }

        /**
        * 编译后
        */
        class P{
          private Object val;
          public void set(Object val){
            this.val = val;
          }
          public Object get(){
            return this.val;
          }
        }

        class C extends P{
          public void set(Date val){
            super.set(val);
          }

          public Date get(){
            return super.get();
          }

          // 编译器自动生成桥接方法
          @Override
          public void set(Object val){
            this.set((Date)val);
          }

          // 编译器自动生成桥接方法
          // Object get() 和 Date get()仅返回值不同，而参数列表相同。在编译时被认为是相同的方法，会报重复定义。
          // 但JVM将返回值+参数列表作为方法签名，因此在运行时被认为是不同的方法。
          @Override
          public Object get(){
            return this.get();
          }
        }

*** 泛型的型变(type variance)——协变(convariance)和逆变(contravariance)
    型变(type variance)
    #+BEGIN_QUOTE
    Type variance refers to the techniques by which we can allow, or not allow, subtyping in our parameterized types.
    #+END_QUOTE
    假设现在存在 ~List<Orange>~ 和 ~List<Fruit>~ 两个类型，它们可以存在以下任一个关系：
    1) 不型变(invariance): ~List<Orange>~ 和 ~List<Fruit>~ 两者没有任何关系；
    2) 协变(covariance): ~List<Orange>~ 为 ~List<Fruit>~ 的子类型；
    3) 逆变(cotravariance): ~List<Fruit>~ 为 ~List<Orange>~ 的子类型；

    *子类(Subclass)* 和 *子类型(Subtype)* 不是一个概念，子类一定是子类型，但子类型不一定是子类。如 ~List<Orange>~ 是 ~List<? extends Fruit>~ 的子类型，但不是子类。

**** Java中的协变
     #+BEGIN_EXAMPLE java
     List<? extends Object> oList;
     List<String> sList = new LinkedList<String>();
     sList.add("hello");
     oList = sList;

     for (int i = 0; i < oList.size(); i++){
       System.out.println(oList.get(i));
     }

     // 编译失败
     // oList.add("123");
     #+END_EXAMPLE
     Java编译器通过限制方法参数列表中包含类型参数的方法调用，保证运行时的类型安全。也就是方法签名如 ~void test(T a)~ ，其中的 ~T~ 会被 ~? extends Object~ 替换掉，即使方法体并没有用到。

     *特点：*
     1) 禁止向泛型类执行写入操作
     2) 允许安全地从泛型类中读取数据

**** Java中的逆变
     #+BEGIN_EXAMPLE java
     List<? super Fruit> superFruitList = new LinkedList<Object>();
     superFruitList.add(new Orange());
     superFruitList.add(new Fruit());

     Object o = superFruitList.get(0);
     Fruit f = (Fruit)superFruitList.get(1);
     #+END_EXAMPLE

     *特点：*
     1) 允许安全地向泛型类执行写入操作
     2) 无法保证运行时从泛型类中读取数据类型的安全性

**** 为什么需要泛型的型变
     在泛型类型安全的基础上，提高API的灵活性。

    若调用的方法能匹配到固定参数重载方法和可变参数重载方法，那么优先匹配固定参数重载方法。
    #+BEGIN_EXAMPLE java
    public class Test{
      public static String concat(String a, String b){
        System.out.println("concat(String a, String b)");
        return a + b;
      }
      public static String concat(String... strs){
        System.out.println("concat(String... strs)");
        StringBuilder sb = new StringBuilder();
        for (String str : strs){
          sb.append(str);
        }
        return sb.toString();
      }

      public static void main(String[] args){
        concat("a", "b"); // concat(String a, String b)
        concat("a", "b", "c"); // concat(String... strs)
      }
    }
    #+END_EXAMPLE

*** 如匹配到多个可变参数重载方法，则会报编译时异常
    #+BEGIN_EXAMPLE java
    public class Test{
      public static void test(String a, Integer... b){
        System.out.println("Integer");
      }
      public static void test(String a, String... b){
        System.out.println("String");
      }

      public static void main(String[] args){
        // test("abc", null); 由于无法判断null的数据类型，因此会抛出编译时异常
        String[] b = null;
        test("abc", b); // String
      }
    }
    #+END_EXAMPLE

*** 反射
    由于反射是在运行时获取类、方法等的元数据，而可变参数在运行时被编译为数组。因此通过反射调用可变参数方法时，可变参数用数组代替。
    #+BEGIN_EXAMPLE java
    public class Test{
      public static void print(String... strs){
        System.out.println(strs[0]);
      }

      public static void main(String[] args){
        Method method = Test.class.getMethod("print", String[].class);
        method.invoke(null, new String[]{"a", "b"}); // 成功
        method.invoke(null, "a", "b"); // 抛出java.lang.IllegalArgumentException: wrong number of arguments
      }
    }
    #+END_EXAMPLE

** 内部类
   作用：
   1) 当要实现的接口方法与当前类的方法签名冲突时，可通过内部类实现接口方法(内部类可访问外部类所有成员)；
   2) 通过内部类继承其它类型，达到多继承的效果；

*** 分类
    1) 成员内部类
       #+BEGIN_EXAMPLE java
       public class Fruit {
         private double price = 100;

         private class InnerPrice implements Price{
           public double getPrice(){
             // discount 在Price接口中定义
             return price * discount;
           }
         }

         public class InnerTimeToMarket extends TTM {
           private double price = 200;

           @Override
           public Date getTTM(){
             // 当内部类和外部类存在同名成员变量/方法时，调用外部类成员变量/方法则要采用如下方式
             // 即内部类拥有指向外部类的引用
             System.out.printf("OuterClass price is: %d", Fruit.this.price);
             System.out.printf("InnerClass price is: %d", price);
             return new Date();
           }
         }

         public Price askPrice(){
           return new InnerPrice();
         }
       }

       // 调用
       Fruit f = new Fruit();
       // 通过private或protected隐藏内部类，调用则只需调用接口或父类约定的方法即可
       Price p = f.askPrice();
       p.getPrice();

       // 通过public暴露内部类，需要先实例化外部类，再实例化其内部类
       Fruit.InnerTimeToMarket ttm = f.new InnerTimeToMarket();
       #+END_EXAMPLE
       - 注意：
         1) 成员内部类不能拥有静态变量
         2) 成员内部类不能拥有静态方法
         3) 成员内部类不能拥有静态内部类
         4) 成员内部类可定义 ~static final~ 常量
    2) 静态内部类
       #+BEGIN_EXAMPLE java
       public static class Fruit {
         private static double price = 100;

         public static class InnerTimeToMarket extends TTM {
           private static double price = 200;

           @Override
           public Date getTTM(){
             // 当内部类和外部类存在同名静态变量/方法时，调用外部类静态变量/方法则要采用如下方式
             System.out.printf("InnerClass price is: %d", Fruit.price);
             System.out.printf("InnerClass price is: %d", price);
             return new Date();
           }
         }
       }

       // 调用
       Fruit.InnerTimeToMarket ttm = new Fruit.InnerTimeToMarket();
       Date time = ttm.getTTM();
       #+END_EXAMPLE
       - 注意：
         1) 静态内部类必须定义在静态类中
         2) 静态内部类中仅能访问外部类的静态变量和静态方法
    3) 局部内部类
       #+BEGIN_EXAMPLE java
       if (true){
         final String name = "john";
         class InnerClass{
           public void print(){
             System.out.println(name);
           }
         }
         InnerClass n = new InnerClass();
       }
       #+END_EXAMPLE
       - 注意
         1) 定义在成员方法的局部内部类，可以访问外部类的所有成员变量和成员方法
         2) 定义在静态方法的局部内部类，可以访问外部类的所有静态变量和静态方法
         3) 局部内部类可以访问方法中的修饰为 ~final~ 的局部变量和方法入参
            - 原理：
              由于局部变量随调用栈帧的释放而被回收，而内部类实例有可能依然可被访问，两者生命周期不一致。而实际上局部变量将作为构造方法入参，参与局部内部类的实例化过程。但存在如下问题，局部变量为基础类型时，Java默认采用传值方式传参。为避免开发人员误会在局部内部类操作的结果会影响到局部变量，因此通过 ~final~ 修饰局部变量，将基本类型的变量标记为常量，引用类型的则直接传递引用本身。
    4) 匿名内部类
       #+BEGIN_EXAMPLE java
       // new interfacename(){} 或 new superclassname(){}
       if (true){
         Fruit f = new Fruit(){
           public double getPrice(){
             return 1d;
           }
         }
       }
       #+END_EXAMPLE

** Java2后出现的集合框架
   集合框架的设计目标：
   1) 提供高性能的集合操作
   2) 对不同类型的集合采用类似的方式操作
   3) 对某个集合的扩展和适应必须简单

   组成：
   1) 接口
      1) Collection
         标识存储一组不唯一、无序的元素。
      2) List
         继承Collection接口，标识存储一组不唯一、有序的元素。
         - 特点
           1) 检索效率高
           2) 插入、删除效率低(会引起其它元素位置改变)
      3) Set
         继承Collection接口，标识存储一组唯一、无序的元素。
         - 特点
           1) 检索效率低
           2) 插入、删除效率高(不会引起其它元素位置改变)
      4) Queue
         继承Collection接口，标识一组遵循FIFO操作的元素。
      5) Map
         标识一系列键值对。
         1) ~Set<K> keySet([V defaultValue])~ 返回可变的键集合，若删除键集合的元素则会自动删除映射对应的条目，若向键集合添加元素，那么需要传入defaultValue作为值的默认值。
      6) SortedMap
         继承Map接口，标识一系列将key按升序排列的键值对。
      7) NavigableMap
         继承SortedMap接口。
      8) Map$Entry
         在Map接口内部定义的，标识一组键值对。
   2) 抽象类/实现类
      1) AbstractCollection
         实现大部分Collection接口的方法。
      2) AbstractList
         继承AbstractCollection和List，并实现大部分List接口的方法。
      3) AbstractSequentialList
         继承AbstractList，提供对集合元素的链式访问而不是随机访问。
      4) LinkedList
         链式数据结构的列表，非线性安全。
         - 特点
           1) 检索效率低
           2) 插入、删除效率高
         - 线性安全改造

           List<String> unsafeList = new LinkedList();

           #+END_EXAMPLE
      5) ArrayList
         大小可变的数组，非线性安全。
         - 特点
           1) 检索效率高
           2) 插入、删除效率低
         - 线性安全改造

      6) Vector
         继承AbstractList
      7) Stack
         继承Vector，提供一组遵循FILO操作集合
      8) AbstractSet
         继承AbstractCollection和Set，并实现大部分Set接口的方法。
      9) HashSet
         存储一组唯一、无序的元素。
      10) LinkedHashSet

      11) TreeSet
          提供排序等功能
      12) AbstractMap
          集成Map接口，实现大部分Map的方法
      13) HashMap
          继承AbstractMap，根据键的HashCode值存储数据，检索效率高，非线性同步
      14) Hashtable
          继承AbstractMap，根据键的HashCode值存储数据，检索效率高，线性同步
      15) SortedMap

      16) TreeMap

      17) WeakHashMap

      18) LinkedHashMap

      19) IdentityHashMap

Class类(类型类)
1) 即所有Java类型(包括数组)的均为Class类的实例。如：
   #+begin_example java
   String instanceof Class<String> == true;
   Integer instanceof Class<Integer> == true;
   Boolean  instanceof Class<Boolean> == true;
   // ......
   #+end_example
2) 接口、8种基本数据类型和void类型，都不是Class类的实例。
3) 8中基本数据类型和void类型的包装类为Class类的实例。

   Class<Integer> clazz = (Class<Integer>) Class.getPrimitiveClass("int");
   #+end_example

5) Class类没有公共的构造函数，不能显式创建一个Class实例。
6) JVM虚拟机加载类后会调用 ~defineClass~ 方法构造出Class实例。JVM采用懒加载方式，当需要使用某个类时才会进行加载和实例化。
7) Class对象必须由 *类加载器* 和 *权限定类名* 来共同确定其在JVM中的唯一性。

获取方式
1) ~类名.class~, 不会触发Class对象的初始化，而是将Class对象的初始化延迟到对运行时静态字段或静态方法的首次引用。

    Type getRawType();

  - ~public static InputStream getSystemResourceAsStream(String name)~ 静态方法调用该方法获取classpath下的资源。

* ClassLoader(类加载器)
  类文件可以存放在文件系统、JAR文件里、远程某个位置或者在内存中动态构建。而类加载器就是负责加载字节流，并且在虚拟机中将它们转化为一个类或接口。

  1) Bootstrap ClassLoader
     C++实现属于虚拟机的一部分，作为Ext ClassLoader的父类加载器，负责jdk_home/lib目录下的核心API(如jre/lib/rt.jar)或-Xbootclasspath指定的jar包装入工作。
     - 代码中无法获取Bootstrap ClassLoader实例 ~String.class.getClassLoader() == null~
  2) System ClassLoader(系统类加载器)
     负责 ~java -classpath/-Djava.class.path~ 所指目录下的类和jar包装入工作。
     - 在Oracle Java中, ~this.getClass().getClassLoader() instanceof URLClassLoader~
  3) Ext ClassLoader(扩展类加载器)
     作为App ClassLoader的父类加载器，负责jdk_home/lib/ext目录下的jar包或-Djava.ext.dirs指定目录下的jar包等"标准库扩展"装入工作。
     - 在Oracle Java中, ~this.getClass().getClassLoader() instanceof URLClassLoader~
  4) App ClassLoader
     负责加载用户类。
  5) Customer ClassLoader
     一般都继承App ClassLoader，用户自定义。

** 类加载器反转(classloader inversion)
   实现动态加载类的方法所用的类加载器，与加载其自身的类加载器不同，称为类加载器反转。
   - 问题示例
     #+BEGIN_EXAMPLE java
     package com.john;

     class T {
       public void load(String name){
         Class.forName(name);
       }
     }

     class Main {
       public void plugin(){
         URL[] urls = {
           new URL("file:///path/to/directory")
         };
         URLClassLoader loader = new URLClassLoader(urls);
         Class<?> tClass = Class.forName("com.john.T", true, loader);
         tClass.getMethod("load").invoke(tClass.newInstance(), "com.john.TT");
       }
     }
     #+END_EXAMPLE
   - 解决办法——上下文类加载器
     #+BEGIN_EXAMPLE java
     package com.john;

     class T {
       public void load(String name){
         Class.forName(name, true, Thread.currentThread().getContextClassLoader());
       }
     }

     class Main {
       public void plugin(){
         URL[] urls = {
           new URL("file:///path/to/directory")
         };
         URLClassLoader loader = new URLClassLoader(urls);
         ClassLoader cl = Thread.currentThread().getContextClassLoader();
         Thread.currentThread().setContextClassLoader(loader);
         Class<?> tClass = Class.forName("com.john.T", true, loader);
         tClass.getMethod("load").invoke(tClass.newInstance(), "com.john.TT");
         Thread.currentThread().setContextClassLoader(cl);
       }
     }
     #+END_EXAMPLE

* TODO JNDI(Java Naming and Directory Interface)
  是一组在Java中访问命名和目录服务的API，作为J2EE的核心技术之一。
  1) 命名服务
     将名称和对象联系起来，让我们可以通过名称访问对象。
  2) 目录服务
     基于命名服务上作增强，使得对象不仅有名称还有属性。

  JNDI组成
  1) 应用编程接口(API)
  2) Naming Manager
  3) 服务提供者接口(SPI,Service Provider Interface)
  4) LDAP服务提供者、NIS服务提供者、文件系统服务提供者等

  优点：通过名称引用Java对象或资源，而不必知道对象或资源的物理位置。

  JNDI现在可访问的命名和目录服务有
  1) DNS
  2) XNam
  3) Novell目录服务
  4) LDAP
  5) CORBA对象服务
  6) 文件系统
  7) Windows的注册表
  8) RMI
  9) DSML v1&v2
  10) NIS

  在J2EE环境下，JNDI的主要用途
  1) 获得Java事务API中的UserTransaction接口的引用
  2) 获得如JDBC、JMS等连接到资源工厂
  3) 在EJB组件中，查找其他EJB组件

  JNDI的概念
  1) Atomic Name(原子名)
     不可分割的名字，如/ETC/FSTAB中的ETC和FSTAB均为Atomic Name
  2) Compound Name(复合名)
     由0或多个Atomic Name组成，如/ETC/FSTAB就是Compound Name
  3) Binding(绑定)
     名字和对象或资源的关联
  4) Context(上下文)
     包含0或多个Binding，如java/MySql中java为Context，而MySql为名字，整体和某个对象或资源关联起来了
  5) Subcontext(子上下文)
     上下文的上下文，如java/ejb/helloBean中ejb为Subcontext
  6) 命名系统
     构成命名空间定义的语法语义
  7) 命名空间
     根据命名系统的语法约定由原子名、复合名、上下文构成的结构，如文件系统的命名空间由目录和文件名组成
  8) 合成名
     由多个命名系统组成的名字，如URL ~http://java.sun.com/product/index.html~
     1) http来自URL scheme-id命名系统
     2) java.sum.com来自DNS命名系统
     3) product、index.html来自文件系统的命名系统

** LDAP(Light Directory Access Protocol)
   相当于简易版本的RDBMS，以树状的层次结构存储数据，适用于读多写少的场景。（优化过读取操作）

*** 概念
    1) Entry
       LDAP基本存储单元,一个DN和一组属性的集合.一个Entry必须包含1个或以上的Objectclass属性.
    2) DN(Distinguished Name)
       标记记录路径的唯一标识。其中Base DN为基准DN，即从哪个DN开始搜索(搜索范围分为Base,One Level和Subtree)。而RDN为叶子节点本身的标识。
       - Base 只搜索baseDN本层的记录
       - One Level 只搜索baseDN直接下级的记录
       - Subtree 搜索baseDN全部下级的记录
    3) DC(Domain Component)
       记录所属区域
    4) OU(Organization Unit)
       组织单元
    5) CN/UID
       记录的名字
    6) Objectclass(对象类)
       LDAP内置的数据类型,每种Objectclass均有自己的数据结构,作为Entry的"模板".如名为"电话簿"的Objectclass内置很多属性(attributes),如身份证号码等.而名为"同学录"Objectclass则内置其他属性.
       1) 属性分类
          1) required attribute
          2) option attribute
          3) 操作属性 独立于Objectclass而存在的属性
       2) 分类
          1) 结构型(Structural),如account,inetOrgPerson,person,organizationUnit
          2) 辅助型(Auxiliary),如extensibleObject
          3) 抽象类(Abstract),不能直接使用的.如top
       3) bind和unbind
          - bind, 增加属性(属性要么隶属于objectclass,要么为操作属性)
          - unbind,移除属性
    7) Operational Attributes
       只读属性
       #+begin_example
       createTimeStamp
       creatorsName
       entryFlags
       federationBoundary
       localEntryID
       modifiersName
       modifyTimeStamp
       structuralObjectClass
       subordinateCount
       subschemaSubentry
       #+end_example
       - Java查询时在ReturnningAttributeNames设置为 ~new String[]{"*","+"}~ 来获取,而设置为 ~null~ 即获取所有非Operational Attributes.

** Lambda表达式
   由编译器提供支持。
   lambda表示函数参数，而带有参数变量的表达式被称为lambda表达式。
   而Lambda表达式在Java中为实现特定 *接口或抽象类* 的 *抽象方法* 的便捷语法。

   #+BEGIN_EXAMPLE java
   Comparator<String> comparator = (String a, String b) -> Integer.compare(a.length(), b.length());
   // 对于函数式接口，方法类型可被推导，因此参数列表可以省略入参类型
   Comparator<String> comparator = (a, b) -> Integer.compare(a.length(), b.length());
   Comparator<String> comparator = (a, b) -> {
     int aLen = a.length();
     int bLen = b.length();

     return Integer.compare(aLen + 1, bLen);
   }

   // 参数列表为空时
   Runnable task = () -> {
     for (int i = 0; i < 1000; i++){
       // do something
     }
   };

   // 仅有一个参数时，可以省略括号
   Predicate<String> predicate = x -> Objects.isNull(x);
   #+END_EXAMPLE

*** 作用域
    Lambda表达式的方法体和所在代码块有着相同的作用域。
    1) Lambda表达式中不能有与局部变量同名的参数或局部变量
    2) this指向所在代码块所属的实例
    3) 只能访问声明为final的自由变量
       - 什么是自由变量：不是从入参传递过来的参数
         #+BEGIN_EXAMPLE java
         final int a = 1;
         Runnable task = () -> System.out.println(a); // a就是自由变量
         #+END_EXAMPLE
       - 什么是闭包(closure)：含自由变量的代码块
       - 增强for循环的变量自动实现final
         #+BEGIN_EXAMPLE java
         for (String arg : args){
           new Thread(() -> System.out.println(arg)).start();
         }
         #+END_EXAMPLE

*** 抛出异常
    若Lambda表达式内执行可能抛出checked exception的操作，那么函数式接口中的抽象方法必须声明该异常，否则编译失败。
    #+BEGIN_EXAMPLE
    public interface Consumer<T>{
      void accept(T t);
    }
    // write可能抛出IOException等异常，但accept方法没有声明异常，因此编译失败

   | 接口                 | 参数类型 | 返回类型 | 抽象方法 | 默认方法                 |

   | Runnable             |          | void     | run      |                          |
   | Supplier<T>          |          | T        | get      |                          |
   | Comsumer<T>          | T        | void     | accept   | andThen                  |
   | BiComsumer<T, U>     | T,U      | void     | accept   | andThen                  |
   | Function<T,R>        | T        | R        | apply    | compose,andThen,identity |
   | BiFunction<T, U, R>  | T, U     | R        | apply    | compose                  |
   | UnaryOperator<T>     | T        | T        | apply    | compose,andThen,identity |
   | BinaryOperator<T>    | T,T      | T        | apply    | andThen,minBy,maxBy      |
   | Predicate<T>         | T        | boolean  | test     | and,or,negate,isEqual    |
   | BiPredicate<T, U>    | T, U     | boolean  | test     | and,or,negate            |
   另外 ~java.util.function~ 中还为基本类型int,long和double提供函数式接口，减少装箱拆箱的损耗。

            if (o == this) rerturn true;
            if (!(o instanceof Test)) return false;
            Test other = (Test)o;
            if (!other.canEqual(this)) return false;
            if (this.getA() == null ? other.getA() != null : !this.getA().equals(other.getA())) return false;
            if (this.getB() == null ? other.getB() != null : !this.getB().equals(other.getB())) return false;

            return result;
          }

          @Override
          public String toString() {
            return "Test(" + this.getA() + ", " + this.getB() + ")";

** 工作原理分析
   JDK5后注解提供两种解析方式：
   1) 运行时解析
      ~@Retention(RUNTIME)~ 标记注解为运行时解析，通过反射获取注解信息(~java.lang.reflect.AnnotatedElement~ 接口)
   2) 编译时解析
      - 两种机制
        1) Annotation Processing Tool
           JDK5引入但JDK7废弃，JDK8直接被删除。被替换的原因：1. API都在com.sum.mirror非标准包中；2. 没有集成到javac中，需要额外运行。
        2) Pluggable Annotation Processing API([[https://jcp.org/en/jsr/detail?id=269][JSR 269]])
           JDK6引入用于替代Annotation Processing Tool.
           - 工作流程
             1) javac对源码进行分析，生活了一棵抽象语法树(AST)
             2) 运行中调用实现了JSR 269 API的Lombok程序
             3) Lombok程序对AST进行处理，根据注解生成树节点
             4) javac根据AST生成字节码文件

*** 核心组件

*** 数据绑定
分为 *简单数据绑定* 和 *完整数据绑定*

**** 简单数据绑定
将JSON字符串转换为Java核心的数据类型，而不用定义POJO对象。
| JSON类型 | Java类型              |
|----------+-----------------------|
| object   | LinkedHashMap         |
| array    | ArrayList             |
| string   | String                |
| number   | Integer, Long, Double |
| boolean  | Boolean               |
| null     | null                  |

#+begin_example java
ObjectMapper objMapper = new ObjectMapper();

// 反序列化
Map<Strting, Object> map = objMapper.readValue("{\"id\": 1, \"name\": \"john\", \"labels\": [1,2], \"dep\": {\"id\": 1}}"
                                               , HashMap<String, Object>.class);
Integer id = (Integer)map.get("id");
String name = (String)map.get("name");
ArrayList<?> labels = (ArrayList<?>)map.get("labels");
Map<String, Object> dep = (Map<String, Object>)map.get("dep");
Integer depId = (Integer)dep.get("id");
Object notExist = map.get("notExist"); // 不存在的返回null

// 序列化
String json = objMapper.writeValueAsString(map);
#+end_example

**** 完整数据绑定
需要定义POJO对象进行JSON的序列化和反序列化，注意：
1) 若JSON字符串中存在属性A，而POJO中没有对应字段，那么反序列化时会抛出UnrecognizedPropertyException异常；
2) 若JSON字符串中缺少POJO的字段，那么序列化时POJO对应的字段将被设置为null。

#+begin_example java
class Person {
    public int id;
    private String name;
    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return this.name;
    }
}

ObjectMapper objMapper = new ObjMapper();
// 反序列化
Person person = objMapper.readValue(jsonString, Person.class);

// 序列化
String json = objMapper.writeValueAsString(person);
#+end_example

*** 树模型
通过get、JsonPointer等操作，适用于获取大JSON中的字段。

#+begin_example java
ObjectMapper objMapper = new ObjMapper();
// 反序列化
JsonNode rootNode = mapper.readTree(jsonString);
// 通过path获取属性值
rootNode.path("name").asText();
// 通过get获取属性值
rootNode.get("name").asText();
// 通过JsonPointer获取嵌套的属性值
JsonPointer pointer = JsonPointer.valueOf("/dep/id");
JsonNode depIdNode = rootNode.at(pointer);
depIdNode.asText();
// 获取数组
JsonNode arrayNode = rootNode.get("labels");
for (int i = 0; i < arrayNode.size(); i++) {
    arrayNode.get(i).asText();
}
JsonNode notExistNode = rootNode.path("notExist");
notExistNode.isMissingNode(); // 返回true
// 当JSON中key存在，但值为null，那么会返回NullNode实例。
JsonNode nullNode = rootNode.path("nullNode");

// 序列化
JsonNode rootNode = mapper.createObjectNode();
((ObjectNode)rootNode).putArray("labels");
ArrayNode labelsNode = (ArrayNode)rootNode.get("labels");
labelsNode.add("1");
labelsNode.add("2");
((ObjectNode)rootNode).put("id", 1);
String json = mapper.writeValueAsString(rootNode);
#+end_example

*** 流式API
通过JsonGenerator序列化，通过JsonParser反序列化。调用底层API所以速度较前两者较快。

*** 注解说明
- ~JsonProperty(String value, int index)~ 标识JSON映射的字段，value指定JSON属性名称(默认为字段名称)，index指定JSON属性的顺序
- ~JsonIgnore~ 标识不被映射的字段
- ~JsonIgnoreProperties~ 类注解，设置多个不被映射的字段
  #+begin_example java
  @JsonIngoreProperties({"p1", "p2"})
  public class P {
      public String p1;
      public String p2;
      public String p3;
  }
  #+end_example
  - ~@JsonIgnoreProperties(ignoreUnknown=true)~ JSON反序列化为POJO时，排除所有没有Getter/Setter的字段。
- ~JsonIngoreType~ 类注解，排除该类的所有字段
- ~JsonPropertyOrder~ 指定JSON属性的顺序
- ~JsonRootName~ 指定JSON根属性的名称
- ~JsonInclude(Include.NON_EMPTY)~ 仅在不为空时才序列化此字段。若字段类型为字符串，则为null或""时就不参与序列化
  1) Include.ALWAYS 默认值,表示总序列化
  2) Include.NON_DEFATULT 表示不序列化
  3) Include.NON_EMPTY 为空字符串或null时不序列化
  4) Include.NON_NULL 为null时不序列化

*** 配置
#+begin_example java
// 美化输出
mapper.enable(SerializationFeature.INDENT_OUTPUT);
// 允许序列化空的POJO类
// （否则会抛出异常）
mapper.disable(SerializationFeature.FAIL_ON_EMPTY_BEANS);
// 把java.util.Date, Calendar输出为数字（时间戳）
mapper.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);

// 在遇到未知属性的时候不抛出异常
mapper.disable(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES);
// 强制JSON 空字符串("")转换为null对象值:
mapper.enable(DeserializationFeature.ACCEPT_EMPTY_STRING_AS_NULL_OBJECT);

// 在JSON中允许C/C++ 样式的注释(非标准，默认禁用)
mapper.configure(JsonParser.Feature.ALLOW_COMMENTS, true);
// 允许没有引号的字段名（非标准）
mapper.configure(JsonParser.Feature.ALLOW_UNQUOTED_FIELD_NAMES, true);
// 允许单引号（非标准）
mapper.configure(JsonParser.Feature.ALLOW_SINGLE_QUOTES, true);
// 强制转义非ASCII字符
mapper.configure(JsonGenerator.Feature.ESCAPE_NON_ASCII, true);
// 将内容包裹为一个JSON属性，属性名由@JsonRootName注解指定
mapper.configure(SerializationFeature.WRAP_ROOT_VALUE, true);

// 非null字段才参与序列化
mapper.setSerializationInclusion(Include.NON_NULL);

Jackson2.9内置支持CSV序列化和反序列化.

CSV介绍
1) 不具备自描述功能，因此接收处理方必须清楚每列表示的数据类型
2) 可以包含非数据部分，如头行
3) 可以以 ~,~, ~|~, 空格, tab 等作为列分隔符
4) 可以以 ~\n~ 或其它字符作为行分隔符

** 定义schema,包含必填项 *列名* 和 *列顺序* ，可选项 *列数据类型*
1) 方式1：通过POJO设置
   #+begin_example java
   public enum Gender {MALE, FEMALE};
    @JsonPropertyOrder({"name", "age", "gender", "verified", "image"})
   public class User {
       public String name;
       public int age;
       public Gender gender; // 对于结构体则需要自行定义序列化和反序列化方法
       public boolean verified;
       public byte[] image;
   }
    CsvMapper mapper = new CsvMapper();
   CsvSchema schema = mapper.schemaFor(User.class);
   #+end_example
2) 方式2：通过fluent API设置
   #+begin_example java
   CsvSchema schema = CsvSchema.builder()
     .addColumn("name")
     .addColumn("age")
     .addColumn("gender")
     .addColumn("verified")
     .addColumn("image")
     .build();
   #+end_example
