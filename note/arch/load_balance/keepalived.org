除了能管理LVS外,还可以作为Nginx/Haproxy/Mysql等的高可用解决方案.
Keepalived通过VRRP协议(Virtual Router Redundacy Protocol, 原目的为解决静态路由单点故障问题)实现IP故障转移(IP overfail)提供高可用功能.

* Windows平台下类似的软件 - NLB
群集最多32台主机

** 网络拓扑图
| 用户代理 |      虚拟IP |        NLB |    IIS ARR | Application Server |
|----------+-------------+------------+------------+--------------------|
|          |             | 192.1.23.2 | 192.1.23.5 |         129.1.23.7 |
| 浏览器   | 192.1.23.20 |       心跳 |            |                    |
|          |             | 192.1.23.3 | 192.1.23.6 |         129.1.23.8 |

** 配置
1) 在192.1.23.[2:3]中安装 *网络负载平衡*
2) 进入 *网络负载平衡管理器*
   1) 新建 *网络负载平衡群集*, 并以一个未使用过的IP作为群集的VIP
   2) 将192.1.23.[2:3]两台IP配置到群集中,并设置优先级.(优先级最高者为Master)

** 筛选模式说明
- 多个主机
  - 无: 即群集存在A/B两台主机,那么请求转发顺序为ABABABABABABABABABABABAB......
  - 单一: 即群集存在A/B两台主机,那么请求转发顺序为AAAAAAAAAAA...... 或 BBBBBBBBBBB......
  - 网络: 根据NLB真实IP的Class C来决定请求转发到对应的NLB
- 单一主机: 优先级高的主机作为Master
- 禁用此端口: 不转发请求到当前主机
